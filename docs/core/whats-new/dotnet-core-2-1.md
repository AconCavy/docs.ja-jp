## <a name="roll-forward"></a><span data-ttu-id="40fe7-150">ロールフォワード</span><span class="sxs-lookup"><span data-stu-id="40fe7-150">Roll forward</span></span>

<span data-ttu-id="40fe7-151">.NET Core 2.0 以降のすべての .NET Core アプリケーションは、システムにインストールされている最新の*マイナー バージョン*に自動的にロールフォワードされます。</span><span class="sxs-lookup"><span data-stu-id="40fe7-151">All .NET Core applications starting with .NET Core 2.0 automatically roll forward to the latest *minor version* installed on a system.</span></span>

<span data-ttu-id="40fe7-152">.NET Core 2.0 以降では、アプリケーションのビルドに使用した .NET Core のバージョンが実行時に存在しない場合、インストールされている .NET Core の最新の*マイナー バージョン*に対してアプリケーションが自動的に実行されます。</span><span class="sxs-lookup"><span data-stu-id="40fe7-152">Starting with .NET Core 2.0, if the version of .NET Core that an application was built with is not present at runtime, the application automatically runs against the latest installed *minor version* of .NET Core.</span></span> <span data-ttu-id="40fe7-153">つまり、アプリケーションが .NET Core 2.0 を使用してビルドされ、ホスト システムにインストールされているのが .NET Core 2.0 ではなく .NET Core 2.1 である場合は、.NET Core 2.1 を使用してアプリケーションが実行されます。</span><span class="sxs-lookup"><span data-stu-id="40fe7-153">In other words, if an application is built with .NET Core 2.0, and .NET Core 2.0 is not present on the host system but .NET Core 2.1 is, the application runs with .NET Core 2.1.</span></span>

> [!IMPORTANT]
> <span data-ttu-id="40fe7-154">このロールフォワードの動作はプレビュー リリースには適用されません。</span><span class="sxs-lookup"><span data-stu-id="40fe7-154">This roll-forward behavior doesn't apply to preview releases.</span></span> <span data-ttu-id="40fe7-155">既定では、メジャー リリースにも適用されませんが、次の設定で変更できます。</span><span class="sxs-lookup"><span data-stu-id="40fe7-155">By default, it also doesn't apply to major releases, but this can be changed with the settings below.</span></span>

<span data-ttu-id="40fe7-156">共有フレームワークの候補なしでロール フォワードの設定を変更することで、この動作を変更できます。</span><span class="sxs-lookup"><span data-stu-id="40fe7-156">You can modify this behavior by changing the setting for the roll-forward on no candidate shared framework.</span></span> <span data-ttu-id="40fe7-157">使用可能な設定は次のとおりです。</span><span class="sxs-lookup"><span data-stu-id="40fe7-157">The available settings are:</span></span>

- <span data-ttu-id="40fe7-158">`0` - マイナー バージョンのロールフォワード動作を無効にします。</span><span class="sxs-lookup"><span data-stu-id="40fe7-158">`0` - disable minor version roll-forward behavior.</span></span> <span data-ttu-id="40fe7-159">この設定では、.NET Core 2.0.0 用にビルドされたアプリケーションが、.NET Core 2.2.0 または .NET Core 3.0.0 ではなく、.NET Core 2.0.1 にロール フォワードされます。</span><span class="sxs-lookup"><span data-stu-id="40fe7-159">With this setting, an application built for .NET Core 2.0.0 will roll forward to .NET Core 2.0.1, but not to .NET Core 2.2.0 or .NET Core 3.0.0.</span></span>
- <span data-ttu-id="40fe7-160">`1` - マイナー バージョンのロールフォワード動作を有効にします。</span><span class="sxs-lookup"><span data-stu-id="40fe7-160">`1` - enable minor version roll-forward behavior.</span></span> <span data-ttu-id="40fe7-161">これが設定の既定値です。</span><span class="sxs-lookup"><span data-stu-id="40fe7-161">This is the default value for the setting.</span></span> <span data-ttu-id="40fe7-162">この設定では、.NET Core 2.0.0 用にビルドされたアプリケーションが、インストールされているバージョンに応じて、.NET Core 2.0.1 または .NET Core 2.2.0 のいずれかにロール フォワードされますが、.NET Core 3.0.0 にはロール フォワードされません。</span><span class="sxs-lookup"><span data-stu-id="40fe7-162">With this setting, an application built for .NET Core 2.0.0 will roll forward to either .NET Core 2.0.1 or .NET Core 2.2.0, depending on which one is installed, but it will not roll forward to .NET Core 3.0.0.</span></span>
- <span data-ttu-id="40fe7-163">`2` - マイナー バージョンとメジャー バージョンのロールフォワード動作を有効にします。</span><span class="sxs-lookup"><span data-stu-id="40fe7-163">`2` - enable minor and major version roll-forward behavior.</span></span> <span data-ttu-id="40fe7-164">設定すると、異なるメジャー バージョンも考慮されるため、.NET Core 2.0.0 用にビルドされたアプリケーションは、.NET Core 3.0.0 にロール フォワードされます。</span><span class="sxs-lookup"><span data-stu-id="40fe7-164">If set, even different major versions are considered, so an application built for .NET Core 2.0.0 will roll forward to .NET Core 3.0.0.</span></span>

<span data-ttu-id="40fe7-165">この設定は、次の 3 つのいずれかの方法で変更できます。</span><span class="sxs-lookup"><span data-stu-id="40fe7-165">You can modify this setting in any of three ways:</span></span>

- <span data-ttu-id="40fe7-168">[.NET Core CLI ツール](../tools/index.md)を使用している場合、次のオプションに目的の値を指定して .NET Core コマンド (`run` など) に追加します。</span><span class="sxs-lookup"><span data-stu-id="40fe7-168">When using [.NET Core CLI tools](../tools/index.md), add the following option with the desired value to a .NET Core command such as `run`:</span></span>


<span data-ttu-id="40fe7-169">修正プログラムのバージョンのロール フォワードは、この設定からは独立しており、任意の潜在的なマイナー バージョンまたはメジャー バージョンのロール フォワードが適用された後に行われます。</span><span class="sxs-lookup"><span data-stu-id="40fe7-169">Patch version roll forward is independent of this setting and is done after any potential minor or major version roll forward is applied.</span></span>
