## <a name="remoting"></a><span data-ttu-id="78b76-121">リモート処理</span><span class="sxs-lookup"><span data-stu-id="78b76-121">Remoting</span></span>

<span data-ttu-id="78b76-122">.NET リモート処理は、問題のあるアーキテクチャであると判断されました。</span><span class="sxs-lookup"><span data-stu-id="78b76-122">.NET Remoting was identified as a problematic architecture.</span></span> <span data-ttu-id="78b76-123">AppDomain 間の通信で使用されていますが、今後はサポートされなくなります。</span><span class="sxs-lookup"><span data-stu-id="78b76-123">It's used for cross-AppDomain communication, which is no longer supported.</span></span> <span data-ttu-id="78b76-124">また、リモート処理にはランタイム サポートが必要で、維持するのに高いコストがかかります。</span><span class="sxs-lookup"><span data-stu-id="78b76-124">Also, Remoting requires runtime support, which is expensive to maintain.</span></span> <span data-ttu-id="78b76-125">これらの理由から、.NET リモート処理は .NET Core でサポートされておらず、また将来サポートが追加される予定もありません。</span><span class="sxs-lookup"><span data-stu-id="78b76-125">For these reasons, .NET Remoting isn't supported on .NET Core, and we don't plan on adding support for it in the future.</span></span>

<span data-ttu-id="78b76-126">プロセス間の通信では、リモート処理に代わる方法として、<xref:System.IO.Pipes> クラスまたは <xref:System.IO.MemoryMappedFiles.MemoryMappedFile> クラスなどのプロセス間通信 (IPC) メカニズムを検討してください。</span><span class="sxs-lookup"><span data-stu-id="78b76-126">For communication across processes, consider inter-process communication (IPC) mechanisms as an alternative to Remoting, such as the <xref:System.IO.Pipes> class or the <xref:System.IO.MemoryMappedFiles.MemoryMappedFile> class.</span></span>

<span data-ttu-id="78b76-127">マシン間では、代替方法としてネットワーク ベースのソリューションを使用してください。</span><span class="sxs-lookup"><span data-stu-id="78b76-127">Across machines, use a network-based solution as an alternative.</span></span> <span data-ttu-id="78b76-128">可能であれば、HTTP などのオーバーヘッドの少ないプレーンテキストのプロトコルを使用してください。</span><span class="sxs-lookup"><span data-stu-id="78b76-128">Preferably, use a low-overhead plain text protocol, such as HTTP.</span></span> <span data-ttu-id="78b76-129">この場合、ASP.NET Core で使用される Web サーバーの [Kestrel Web Server](/aspnet/core/fundamentals/servers/kestrel) も選択できます。</span><span class="sxs-lookup"><span data-stu-id="78b76-129">The [Kestrel web server](/aspnet/core/fundamentals/servers/kestrel), the web server used by ASP.NET Core, is an option here.</span></span> <span data-ttu-id="78b76-130">また、ネットワーク ベースのマシン間のシナリオとして、<xref:System.Net.Sockets> の使用も検討してください。</span><span class="sxs-lookup"><span data-stu-id="78b76-130">Also, consider using <xref:System.Net.Sockets> for network-based, cross-machine scenarios.</span></span> <span data-ttu-id="78b76-131">その他のオプションについては、[.NET オープン ソース開発者プロジェクト: メッセージング](https://github.com/Microsoft/dotnet/blob/master/dotnet-developer-projects.md#messaging)に関する記事をご覧ください。</span><span class="sxs-lookup"><span data-stu-id="78b76-131">For more options, see [.NET Open Source Developer Projects: Messaging](https://github.com/Microsoft/dotnet/blob/master/dotnet-developer-projects.md#messaging).</span></span>

## <a name="code-access-security-cas"></a><span data-ttu-id="78b76-132">CAS (コード アクセス セキュリティ)</span><span class="sxs-lookup"><span data-stu-id="78b76-132">Code Access Security (CAS)</span></span>

<span data-ttu-id="78b76-133">サンド ボックスは、マネージド アプリケーションやライブラリが使用または実行するリソースの制限を、ランタイムまたはフレームワークに依存しています。これは [.NET Framework ではサポートされていない](../../framework/misc/code-access-security.md)ため、.NET Core でもサポートされていません。</span><span class="sxs-lookup"><span data-stu-id="78b76-133">Sandboxing, which relies on the runtime or the framework to constrain which resources a managed application or library uses or runs, [isn't supported on .NET Framework](../../framework/misc/code-access-security.md) and therefore is also not supported on .NET Core.</span></span> <span data-ttu-id="78b76-134">.NET Framework やランタイムでは、特権の昇格が発生するケースが多すぎるため、このまま CAS をセキュリティ境界と見なすことはできません。</span><span class="sxs-lookup"><span data-stu-id="78b76-134">There are too many cases in the .NET Framework and the runtime where an elevation of privileges occurs to continue treating CAS as a security boundary.</span></span> <span data-ttu-id="78b76-135">さらに、CAS は実装が複雑化しており、その使用を予定していないアプリケーションでは、多くの場合で正確性のパフォーマンスに影響します。</span><span class="sxs-lookup"><span data-stu-id="78b76-135">In addition, CAS makes the implementation more complicated and often has correctness-performance implications for applications that don't intend to use it.</span></span>

<span data-ttu-id="78b76-136">仮想化、コンテナー、ユーザー アカウントなど、オペレーティング システムが提供するセキュリティ境界を使用して、最小限の特権セットでプロセスを実行します。</span><span class="sxs-lookup"><span data-stu-id="78b76-136">Use security boundaries provided by the operating system, such as virtualization, containers, or user accounts, for running processes with the minimum set of privileges.</span></span>

<span data-ttu-id="78b76-140">仮想化、コンテナー、ユーザー アカウントなど、オペレーティング システムが提供するセキュリティ境界を使用して、最低限の特権セットでプロセスを実行します。</span><span class="sxs-lookup"><span data-stu-id="78b76-140">Use security boundaries provided by the operating system, such as virtualization, containers, or user accounts, for running processes with the least set of privileges.</span></span>
