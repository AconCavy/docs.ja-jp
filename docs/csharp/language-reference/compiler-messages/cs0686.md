# <a name="compiler-error-cs0686"></a><span data-ttu-id="1c179-103">コンパイラ エラー CS0686</span><span class="sxs-lookup"><span data-stu-id="1c179-103">Compiler Error CS0686</span></span>

<span data-ttu-id="1c179-104">アクセサー 'accessor' は、インターフェイス メンバー 'member' を型 'type' に対して実装できません。</span><span class="sxs-lookup"><span data-stu-id="1c179-104">Accessor 'accessor' cannot implement interface member 'member' for type 'type'.</span></span> <span data-ttu-id="1c179-105">明示的なインターフェイスの実装を使用します。</span><span class="sxs-lookup"><span data-stu-id="1c179-105">Use an explicit interface implementation.</span></span>  
  
 <span data-ttu-id="1c179-106">推奨事項: このエラーは、プロパティまたはイベントと関連付けられている自動生成メソッドと名前が競合するメソッドを含むインターフェイスを実装すると発生する可能性があります。</span><span class="sxs-lookup"><span data-stu-id="1c179-106">Suggested: This error can occur when implementing an interface that contains method names which conflict with the auto-generated methods associated with a property or event.</span></span> <span data-ttu-id="1c179-107">プロパティの get/set メソッドは、get_property および set_property として生成されます。また、イベントの add/remove メソッドが add_event および remove_event として生成されます。</span><span class="sxs-lookup"><span data-stu-id="1c179-107">The get/set methods for properties are generated as get_property and set_property, and the add/remove methods for events are generated as add_event and remove_event.</span></span> <span data-ttu-id="1c179-108">インターフェイスにこれらと同じ名前のメソッドが存在すると、競合が発生します。</span><span class="sxs-lookup"><span data-stu-id="1c179-108">If an interface contains either of these methods, a conflict occurs.</span></span> <span data-ttu-id="1c179-109">このエラーを回避するには、明示的なインターフェイスの実装を使ってメソッドを実装します。</span><span class="sxs-lookup"><span data-stu-id="1c179-109">To avoid this error, implement the methods using an explicit interface implementation.</span></span> <span data-ttu-id="1c179-110">そのためには、関数を次のように指定します。</span><span class="sxs-lookup"><span data-stu-id="1c179-110">To do this, specify the function as:</span></span>  
  
```csharp  
Interface.get_property() { /* */ }  
Interface.set_property() { /* */ }  
```  
  
## <a name="example"></a><span data-ttu-id="1c179-111">例</span><span class="sxs-lookup"><span data-stu-id="1c179-111">Example</span></span>  

 <span data-ttu-id="1c179-112">次の例では CS0686 が生成されます。</span><span class="sxs-lookup"><span data-stu-id="1c179-112">The following sample generates CS0686:</span></span>  
  
```csharp  
// CS0686.cs  
interface I  
{  
    int get_P();  
}  
  
class C : I  
{  
    public int P  
    {  
        get { return 1; }  // CS0686  
    }  
}  
// But the following is valid:  
class D : I  
{  
    int I.get_P() { return 1; }  
    public static void Main() {}  
}  
```  
  
## <a name="example"></a><span data-ttu-id="1c179-113">例</span><span class="sxs-lookup"><span data-stu-id="1c179-113">Example</span></span>  

 <span data-ttu-id="1c179-114">このエラーは、イベントの宣言で発生する場合もあります。</span><span class="sxs-lookup"><span data-stu-id="1c179-114">This error can also occur when declaring events.</span></span>  <span data-ttu-id="1c179-115">イベントの構造では `add_event` および `remove_event` というメソッドが自動的に生成されるため、次の例のように、インターフェイス内の同じ名前のメソッドと競合する可能性があります。</span><span class="sxs-lookup"><span data-stu-id="1c179-115">The event construct automatically generates the `add_event` and `remove_event` methods, which could conflict with the methods of the same name in an interface, as in the following sample:</span></span>  
  
```csharp  
// CS0686b.cs  
using System;  
  
interface I  
{  
    void add_OnMyEvent(EventHandler e);  
}  
  
class C : I  
{  
    public event EventHandler OnMyEvent  
    {  
        add { }  // CS0686  
        remove { }  
    }  
}  
  
// Correct (using explicit interface implementation):  
class D : I  
{  
    void I.add_OnMyEvent(EventHandler e) {}  
    public static void Main() {}  
}  
```
