# <a name="compiler-error-cs1009"></a><span data-ttu-id="56bd9-103">コンパイラ エラー CS1009</span><span class="sxs-lookup"><span data-stu-id="56bd9-103">Compiler Error CS1009</span></span>

<span data-ttu-id="56bd9-104">認識できないエスケープ シーケンスです</span><span class="sxs-lookup"><span data-stu-id="56bd9-104">Unrecognized escape sequence</span></span>  
  
 <span data-ttu-id="56bd9-105">予期しない文字は [文字列](../builtin-types/reference-types.md#the-string-type)の円記号 (\\) に従います。</span><span class="sxs-lookup"><span data-stu-id="56bd9-105">An unexpected character follows a backslash (\\) in a [string](../builtin-types/reference-types.md#the-string-type).</span></span> <span data-ttu-id="56bd9-106">コンパイラは、有効なエスケープ文字のうち 1 つを要求します。</span><span class="sxs-lookup"><span data-stu-id="56bd9-106">The compiler expects one of the valid escape characters.</span></span> <span data-ttu-id="56bd9-107">詳細については、「[文字のエスケープ](../../../standard/base-types/character-escapes-in-regular-expressions.md)」を参照してください。</span><span class="sxs-lookup"><span data-stu-id="56bd9-107">For more information, see [Character Escapes](../../../standard/base-types/character-escapes-in-regular-expressions.md).</span></span>  
  
 <span data-ttu-id="56bd9-108">次の例では CS1009 が生成されます。</span><span class="sxs-lookup"><span data-stu-id="56bd9-108">The following sample generates CS1009.</span></span>  
  
```csharp  
// CS1009-a.cs  
class MyClass  
{  
   static void Main()  
   {  
      // The following line causes CS1009.  
      string a = "\m";
      // Try the following line instead.  
      // string a = "\t";  
   }  
}  
```  
  
 <span data-ttu-id="56bd9-109">このエラーの一般的な原因は、次の例のように、ファイル名に円記号が使用されていることです。</span><span class="sxs-lookup"><span data-stu-id="56bd9-109">A common cause of this error is using the backslash character in a file name, as the following example shows.</span></span>  
  
```csharp  
string filename = "c:\myFolder\myFile.txt";  
```  
  
 <span data-ttu-id="56bd9-110">このエラーを解決するには、次の例のように「\\\\」または @-quoted のリテラル文字列を使用します。</span><span class="sxs-lookup"><span data-stu-id="56bd9-110">To resolve this error, use "\\\\" or the @-quoted string literal, as the following example shows.</span></span>  
  
```csharp  
// CS1009-b.cs  
class MyClass  
{  
   static void Main()  
   {  
      // The following line causes CS1009.  
      string filename = "c:\myFolder\myFile.txt";
      // Try one of the following lines instead.  
      // string filename = "c:\\myFolder\\myFile.txt";  
      // string filename = @"c:\myFolder\myFile.txt";  
   }  
}  
```
