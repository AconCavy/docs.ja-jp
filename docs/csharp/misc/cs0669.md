# <a name="compiler-error-cs0669"></a><span data-ttu-id="2ce17-103">コンパイラ エラー CS0669</span><span class="sxs-lookup"><span data-stu-id="2ce17-103">Compiler Error CS0669</span></span>

<span data-ttu-id="2ce17-104">ComImport 属性を持つクラスは、ユーザー定義のコンストラクターを持つことはできません。</span><span class="sxs-lookup"><span data-stu-id="2ce17-104">A class with the ComImport attribute cannot have a user-defined constructor.</span></span>

<span data-ttu-id="2ce17-105">[ComImport](xref:System.Runtime.InteropServices.ComImportAttribute) クラスのコンストラクターは、共通言語ランタイムの COM 相互運用層に用意されています。</span><span class="sxs-lookup"><span data-stu-id="2ce17-105">The COM interop layer in the common language runtime supplies the constructor for [ComImport](xref:System.Runtime.InteropServices.ComImportAttribute) classes.</span></span> <span data-ttu-id="2ce17-106">そのため、COM オブジェクトは、実行時にマネージド オブジェクトとして使用できます。</span><span class="sxs-lookup"><span data-stu-id="2ce17-106">Consequently, a COM object can be used as a managed object in the runtime.</span></span>

<span data-ttu-id="2ce17-107">次の例では CS0669 が生成されます。</span><span class="sxs-lookup"><span data-stu-id="2ce17-107">The following sample generates CS0669:</span></span>

```csharp
// CS0669.cs
using System.Runtime.InteropServices;
[ComImport, Guid("00000000-0000-0000-0000-000000000001")]
class TestClass
{
   TestClass()   // CS0669, delete constructor to resolve
   {
   }

   public static void Main()
   {
   }
}
```
