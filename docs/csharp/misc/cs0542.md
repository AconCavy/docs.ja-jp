# <a name="compiler-error-cs0542"></a><span data-ttu-id="ff656-103">コンパイラ エラー CS0542</span><span class="sxs-lookup"><span data-stu-id="ff656-103">Compiler Error CS0542</span></span>
<span data-ttu-id="ff656-104">'user-defined type' : メンバー名をそれを囲む型の名前と同じにすることはできません</span><span class="sxs-lookup"><span data-stu-id="ff656-104">'user-defined type' : member names cannot be the same as their enclosing type</span></span>  
  
 <span data-ttu-id="ff656-105">クラスまたは構造体のメンバーは、メンバーがコンストラクターでない限り、クラスまたは構造体と同じ名前にすることはできません。</span><span class="sxs-lookup"><span data-stu-id="ff656-105">The members of a class or struct cannot have the same name as the class or struct, unless the member is a constructor.</span></span>  
  
 <span data-ttu-id="ff656-106">次の例では CS0542 が生成されます。</span><span class="sxs-lookup"><span data-stu-id="ff656-106">The following sample generates CS0542:</span></span>

  
 <span data-ttu-id="ff656-107">このエラーは、コンストラクターに誤って戻り値の型を指定したことにより、コンストラクターが事実上、通常のメソッドとなる場合に発生する可能性があります。</span><span class="sxs-lookup"><span data-stu-id="ff656-107">This error might be caused if you inadvertently put a return type on a constructor, which in effect makes it into an ordinary method.</span></span> <span data-ttu-id="ff656-108">次の例では、 `F` が戻り値の型を含むことから、コンストラクターではなくメソッドとなるため、CS0542 が生成されます。</span><span class="sxs-lookup"><span data-stu-id="ff656-108">The following example generates CS0542 because `F` is a method, not a constructor, because it has a return type:</span></span>  
  
```csharp

{  
   // Remove void from F() to resolve the problem.  
   void F()   // CS0542, same name as the class  
   {  
   }  
}  
  
class MyClass  
{  
   public static void Main()  
   {  
   }  
}  
```
