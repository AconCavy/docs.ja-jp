<span data-ttu-id="13187-104">型パラメーター 'Type Parameter Name 1' に 'struct' 制約があるため、'Type Parameter Name 1' を 'Type Parameter Name 2' の制約として使用することはできません</span><span class="sxs-lookup"><span data-stu-id="13187-104">Type parameter 'Type Parameter Name 1' has the 'struct' constraint so 'Type Parameter Name 1' cannot be used as a constraint for 'Type Parameter Name 2'</span></span>  
  
 <span data-ttu-id="13187-105">値型制約が暗黙的にシールされるため、これらの制約を 2 番目の型パラメーターで制約として使用することはできません。</span><span class="sxs-lookup"><span data-stu-id="13187-105">Value type constraints are implicitly sealed so those constraints cannot be used as constraints on a second type parameter.</span></span> <span data-ttu-id="13187-106">これは、値型をオーバーライドできないためです。</span><span class="sxs-lookup"><span data-stu-id="13187-106">This is because value types cannot be overridden.</span></span> <span data-ttu-id="13187-107">このエラーを解決するには、値型制約を最初の型パラメーターによって間接的に配置するのではなく、2 番目の型パラメーターに直接配置します。</span><span class="sxs-lookup"><span data-stu-id="13187-107">To resolve this error, put a value type constraint directly on the second type parameter, instead of doing so indirectly by means of the first type parameter.</span></span>  
  
## <a name="example"></a><span data-ttu-id="13187-108">例</span><span class="sxs-lookup"><span data-stu-id="13187-108">Example</span></span>  
 <span data-ttu-id="13187-109">次の例では CS0456 が生成されます。</span><span class="sxs-lookup"><span data-stu-id="13187-109">The following sample generates CS0456.</span></span>  
  
```csharp  
// CS0456.cs  
// compile with: /target:library  
public class GenericsErrors  
{  
   public class G5<T> where T : struct  
   {  
      public class N<U> where U : T {}   // CS0456  
      public class N2<U> where U : struct {}   // OK  
   }  
}  
```
