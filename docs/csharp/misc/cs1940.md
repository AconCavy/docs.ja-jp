<span data-ttu-id="64b4e-103">ソース型 'type' に対してクエリ パターンの複数の実装が見つかりました。</span><span class="sxs-lookup"><span data-stu-id="64b4e-103">Multiple implementations of the query pattern were found for source type 'type'.</span></span> <span data-ttu-id="64b4e-104">'method' の呼び出しがあいまいです。</span><span class="sxs-lookup"><span data-stu-id="64b4e-104">Ambiguous call to 'method'.</span></span>  
  
 <span data-ttu-id="64b4e-105">このエラーは、クエリ メソッドの実装が複数定義されていて、クエリに使用する最適な実装をコンパイラが判断できない場合に発生します。</span><span class="sxs-lookup"><span data-stu-id="64b4e-105">This error is generated when multiple implementations of a query method are defined and the compiler cannot disambiguate which one is best to use for the query.</span></span> <span data-ttu-id="64b4e-106">次の例では、 `Select` の両方のバージョンが入力パラメーターとして `int` を受け取り、戻り値として `int` を使用するため、両方とも同じシグネチャを使用することになります。</span><span class="sxs-lookup"><span data-stu-id="64b4e-106">In the following example, both versions of `Select` have the same signature, because they both accept one `int` as an input parameter and have `int` as a return value.</span></span>

## <a name="example"></a><span data-ttu-id="64b4e-109">例</span><span class="sxs-lookup"><span data-stu-id="64b4e-109">Example</span></span>  

 <span data-ttu-id="64b4e-110">次のコードでは CS1940 が生成されます。</span><span class="sxs-lookup"><span data-stu-id="64b4e-110">The following code generates CS1940:</span></span>  
  
```csharp  
// cs1940.cs  
using System; //must include explicitly for types defined in 3.5  
class Test  
{  
    public delegate int Dele(int x);  
    int num = 0;  
    public int Select(Func<int, int> d)  
    {  
        return d(this.num);  
    }  
    public int Select(Dele d) // CS1940  
    {  
        return d(this.num) + 1;  
    }  
    public static void Main()  
    {  
        var q = from x in new Test()  
        select x;  
    }  
}  
```
