# <a name="compiler-error-cs0012"></a><span data-ttu-id="7e338-103">コンパイラ エラー CS0012</span><span class="sxs-lookup"><span data-stu-id="7e338-103">Compiler Error CS0012</span></span>

<span data-ttu-id="7e338-104">型 'type' は、参照されていないアセンブリに定義されています。</span><span class="sxs-lookup"><span data-stu-id="7e338-104">The type 'type' is defined in an assembly that is not referenced.</span></span> <span data-ttu-id="7e338-105">アセンブリ 'assembly' に参照を追加する必要があります。</span><span class="sxs-lookup"><span data-stu-id="7e338-105">You must add a reference to assembly 'assembly'.</span></span>  
  
 <span data-ttu-id="7e338-106">参照される型の定義が見つかりませんでした。</span><span class="sxs-lookup"><span data-stu-id="7e338-106">The definition for a referenced type was not found.</span></span> <span data-ttu-id="7e338-107">このエラーは、必要な .DLL ファイルがコンパイル時にインクルードされていない場合に発生することがあります。</span><span class="sxs-lookup"><span data-stu-id="7e338-107">This could occur if a required .DLL file is not included in the compilation.</span></span> <span data-ttu-id="7e338-108">詳細については、「[ [参照の追加] ダイアログボックス](/visualstudio/ide/how-to-add-or-remove-references-by-using-the-reference-manager) 」と「 [-Reference (C# コンパイラオプション)](../language-reference/compiler-options/reference-compiler-option.md)」を参照してください。</span><span class="sxs-lookup"><span data-stu-id="7e338-108">For more information, see [Add Reference Dialog Box](/visualstudio/ide/how-to-add-or-remove-references-by-using-the-reference-manager) and [-reference (C# Compiler Options)](../language-reference/compiler-options/reference-compiler-option.md).</span></span>  
  
 <span data-ttu-id="7e338-109">次のコンパイル シーケンスは CS0012 になります。</span><span class="sxs-lookup"><span data-stu-id="7e338-109">The following sequence of compilations will result in CS0012:</span></span>  
  
```csharp  
// cs0012a.cs  
// compile with: /target:library  
public class A {}  
```  
  
 <span data-ttu-id="7e338-110">次のことを行います。</span><span class="sxs-lookup"><span data-stu-id="7e338-110">Then:</span></span>  
  
```csharp  
// cs0012b.cs  
// compile with: /target:library /reference:cs0012a.dll  
public class B  
{  
   public static A f()  
   {  
      return new A();  
   }  
}  
```  
  
 <span data-ttu-id="7e338-111">次のことを行います。</span><span class="sxs-lookup"><span data-stu-id="7e338-111">Then:</span></span>  
  
```csharp  
// cs0012c.cs  
// compile with: /reference:cs0012b.dll  
class C  
{  
   public static void Main()  
   {  
      object o = B.f();   // CS0012  
   }  
}  
```  
  
 <span data-ttu-id="7e338-112">この CS0012 は、 `/reference:cs0012b.dll;cs0012a.dll`を使用してコンパイルするか、Visual Studio で [Add Reference Dialog Box](/visualstudio/ide/how-to-add-or-remove-references-by-using-the-reference-manager) を使用して cs0012b.dll と cs0012a.dll への参照を追加することで解決できる場合があります。</span><span class="sxs-lookup"><span data-stu-id="7e338-112">You could resolve this CS0012 by compiling with `/reference:cs0012b.dll;cs0012a.dll`, or in Visual Studio by using the [Add Reference Dialog Box](/visualstudio/ide/how-to-add-or-remove-references-by-using-the-reference-manager) to add a reference to cs0012a.dll in addition to cs0012b.dll.</span></span>
