<span data-ttu-id="b59f9-103">'method1' はリンク要求を含んでいますが、リンク要求を含んでいない 'method2' をオーバーライドまたは実装します。</span><span class="sxs-lookup"><span data-stu-id="b59f9-103">'method1' has a link demand, but overrides or implements 'method2' which does not have a link demand.</span></span> <span data-ttu-id="b59f9-104">セキュリティに問題が発生する可能性があります。</span><span class="sxs-lookup"><span data-stu-id="b59f9-104">A security hole may exist.</span></span>

## <a name="example"></a><span data-ttu-id="b59f9-108">例</span><span class="sxs-lookup"><span data-stu-id="b59f9-108">Example</span></span>  
 <span data-ttu-id="b59f9-109">次の例では CS0688 が生成されます。</span><span class="sxs-lookup"><span data-stu-id="b59f9-109">The following sample generates CS0688.</span></span> <span data-ttu-id="b59f9-110">基底クラスを変更せずにこの警告を解決するには、オーバーライドするメソッドからセキュリティ属性を削除します。</span><span class="sxs-lookup"><span data-stu-id="b59f9-110">To resolve the warning without modifying the base class, remove the security attribute from the overriding method.</span></span> <span data-ttu-id="b59f9-111">この操作によって、セキュリティ上の問題は解決しません。</span><span class="sxs-lookup"><span data-stu-id="b59f9-111">This will not solve the security problem.</span></span>  
  
```csharp  
// CS0688.cs  
// compile with: /W:1  
using System;  
using System.Security.Permissions;  
  
class Base
{  
    //Uncomment the following line to close the security hole  
    //[FileIOPermission(SecurityAction.LinkDemand, All=@"C:\\")]  
    public virtual void DoScaryFileStuff()  
    {  
    }  
}  
  
class Derived: Base  
{  
    [FileIOPermission(SecurityAction.LinkDemand, All=@"C:\\")] // CS0688  
    public override void DoScaryFileStuff()  
    {  
    }  
    static void Main()  
    {  
    }  
}  
```
