# <a name="compiler-warning-level-3-cs0414"></a><span data-ttu-id="5764b-103">コンパイラの警告 (レベル 3) CS0414</span><span class="sxs-lookup"><span data-stu-id="5764b-103">Compiler Warning (level 3) CS0414</span></span>

<span data-ttu-id="5764b-104">private フィールド 'field' は割り当てられていますが、その値が使用されていません</span><span class="sxs-lookup"><span data-stu-id="5764b-104">The private field 'field' is assigned but its value is never used</span></span>

<span data-ttu-id="5764b-105">この警告は、コンパイラで、変数が参照されていないことを確認できるいくつかのシナリオで発生する可能性があります。</span><span class="sxs-lookup"><span data-stu-id="5764b-105">This warning can occur in several scenarios in which the compiler can verify that a variable is never referenced:</span></span>

- <span data-ttu-id="5764b-109">private または内部の静的フィールドの初期化子にのみ、定数値が割り当てられています。</span><span class="sxs-lookup"><span data-stu-id="5764b-109">A private or internal static field is assigned a constant value only in the initializer.</span></span> <span data-ttu-id="5764b-110">フィールドを const に変更することを検討してください。</span><span class="sxs-lookup"><span data-stu-id="5764b-110">Consider changing the field to a const.</span></span>

- <span data-ttu-id="5764b-111">private または内部フィールドに定数値が割り当てられており、#ifdef ディレクティブによって除外されたブロック内でのみ使用されています。</span><span class="sxs-lookup"><span data-stu-id="5764b-111">A private or internal field is assigned constant values and only used in blocks that are excluded by #ifdef directives.</span></span> <span data-ttu-id="5764b-112">#ifdef ブロック内にフィールドを配置することを検討してください。</span><span class="sxs-lookup"><span data-stu-id="5764b-112">Consider putting the field inside the #ifdef block.</span></span>

- <span data-ttu-id="5764b-113">private または内部フィールドの複数の場所に定数値が割り当てられていますが、それ以外はアクセスされていません。</span><span class="sxs-lookup"><span data-stu-id="5764b-113">A private or internal field is assigned constant values in multiple locations but not otherwise accessed.</span></span> <span data-ttu-id="5764b-114">フィールドが不要な場合は、削除することを検討してください。</span><span class="sxs-lookup"><span data-stu-id="5764b-114">If you do not need the field, consider removing it.</span></span> <span data-ttu-id="5764b-115">削除しない場合は、何らかの適切な方法でこのフィールドを使用します。</span><span class="sxs-lookup"><span data-stu-id="5764b-115">Otherwise, use it in some appropriate way.</span></span>

<span data-ttu-id="5764b-116">その他の場合、または提案されている回避策が受け入れ不可の場合は、#pragma 0414 を使用します。</span><span class="sxs-lookup"><span data-stu-id="5764b-116">In other situations, or where the suggested workaround is not acceptable, use #pragma 0414.</span></span>

<span data-ttu-id="5764b-117">次の例では、CS0414 が生成される仕組みの 1 つを示しています。</span><span class="sxs-lookup"><span data-stu-id="5764b-117">The following sample shows one way in which CS0414 will be generated:</span></span>

```csharp

{
   private int i = 1;  // CS0414

   public static void Main()
   { }
```
