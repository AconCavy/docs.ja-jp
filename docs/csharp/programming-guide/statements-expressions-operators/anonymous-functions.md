<span data-ttu-id="62d03-103">匿名関数は、デリゲート型が必要とされる任意の場所で使用できる "インライン" のステートメントまたは式です。</span><span class="sxs-lookup"><span data-stu-id="62d03-103">An anonymous function is an "inline" statement or expression that can be used wherever a delegate type is expected.</span></span> <span data-ttu-id="62d03-104">名前付きデリゲートを初期化するときや、メソッドのパラメーターとして名前付きデリゲート型の代わりに渡したりするときに使用できます。</span><span class="sxs-lookup"><span data-stu-id="62d03-104">You can use it to initialize a named delegate or pass it instead of a named delegate type as a method parameter.</span></span>

<span data-ttu-id="62d03-105">[ラムダ式](lambda-expressions.md)または[匿名メソッド](../../language-reference/operators/delegate-operator.md)を使って匿名関数を作成できます。</span><span class="sxs-lookup"><span data-stu-id="62d03-105">You can use a [lambda expression](lambda-expressions.md) or an [anonymous method](../../language-reference/operators/delegate-operator.md) to create an anonymous function.</span></span> <span data-ttu-id="62d03-106">ラムダ式を使った方がより簡潔で表現性に優れた方法でインライン コードを記述できるため、こちらを使うことをお勧めします。</span><span class="sxs-lookup"><span data-stu-id="62d03-106">We recommend using lambda expressions as they provide more concise and expressive way to write inline code.</span></span> <span data-ttu-id="62d03-107">匿名メソッドとは異なり、一部の型のラムダ式は式ツリー型に変換することができます。</span><span class="sxs-lookup"><span data-stu-id="62d03-107">Unlike anonymous methods, some types of lambda expressions can be converted to the expression tree types.</span></span>
