## <a name="c-language"></a><span data-ttu-id="123d2-107">C# 言語</span><span class="sxs-lookup"><span data-stu-id="123d2-107">C# language</span></span>

<span data-ttu-id="123d2-108">C# の構文は表現力が豊かですが、単純ですぐに覚えることができます。</span><span class="sxs-lookup"><span data-stu-id="123d2-108">C# syntax is highly expressive, yet it's also simple and easy to learn.</span></span> <span data-ttu-id="123d2-109">C、C++、Java、または JavaScript に慣れていれば、C# の中かっこ ({}) 構文をすぐに理解できます。</span><span class="sxs-lookup"><span data-stu-id="123d2-109">The curly brace syntax of C# will be instantly recognizable to anyone familiar with C, C++, Java or JavaScript.</span></span> <span data-ttu-id="123d2-110">これらの言語のいずれかを理解している開発者は、一般に、短期間で C# で生産的に作業できるようになります。</span><span class="sxs-lookup"><span data-stu-id="123d2-110">Developers who know any of these languages are typically able to work productively in C# within a short time.</span></span> <span data-ttu-id="123d2-111">C# には、null 許容型、デリゲート、ラムダ式、パターン マッチング、安全なダイレクト メモリ アクセスなどの強力な機能が実装されています。</span><span class="sxs-lookup"><span data-stu-id="123d2-111">C# provides powerful features such as nullable types, delegates, lambda expressions, pattern matching, and safe direct memory access.</span></span> <span data-ttu-id="123d2-112">C# は、タイプ セーフとパフォーマンスの向上を実現するジェネリック メソッドと型をサポートします。</span><span class="sxs-lookup"><span data-stu-id="123d2-112">C# supports generic methods and types, which provide increased type safety and performance.</span></span> <span data-ttu-id="123d2-113">C# は反復子もサポートしているため、クライアント コードのカスタム動作をコレクション クラスの実装側が定義できます。</span><span class="sxs-lookup"><span data-stu-id="123d2-113">C# provides iterators, which enable implementers of collection classes to define custom behaviors for client code.</span></span> <span data-ttu-id="123d2-114">統合言語クエリ (LINQ) 式により、厳密に型指定されたクエリは高度な言語構成要素になります。</span><span class="sxs-lookup"><span data-stu-id="123d2-114">Language-Integrated Query (LINQ) expressions make the strongly typed query a first-class language construct.</span></span>

<span data-ttu-id="123d2-115">C# は、オブジェクト指向言語として、カプセル化、継承、およびポリモーフィズムの概念をサポートしています。</span><span class="sxs-lookup"><span data-stu-id="123d2-115">As an object-oriented language, C# supports the concepts of encapsulation, inheritance, and polymorphism.</span></span> <span data-ttu-id="123d2-116">親クラスから直接継承できるクラスは 1 つのみですが、クラスで実装できるインターフェイスの数は任意です。</span><span class="sxs-lookup"><span data-stu-id="123d2-116">A class may inherit directly from one parent class, but it may implement any number of interfaces.</span></span> <span data-ttu-id="123d2-117">親クラスの仮想メソッドをオーバーライドする場合、誤って再定義しないように、`override` キーワードを指定する必要があります。</span><span class="sxs-lookup"><span data-stu-id="123d2-117">Methods that override virtual methods in a parent class require the `override` keyword as a way to avoid accidental redefinition.</span></span> <span data-ttu-id="123d2-118">C# の構造体はコンパクトなクラスのようなものです。インターフェイスを実装できるスタック割り当て型ですが、継承はサポートされていません。</span><span class="sxs-lookup"><span data-stu-id="123d2-118">In C#, a struct is like a lightweight class; it's a stack-allocated type that can implement interfaces but doesn't support inheritance.</span></span> <span data-ttu-id="123d2-119">C# には、主にデータ値を格納する目的を持つクラス型であるレコードも用意されています。</span><span class="sxs-lookup"><span data-stu-id="123d2-119">C# also provides records, which are class types whose purpose is primarily storing data values.</span></span>

<span data-ttu-id="123d2-120">C# には、次のような革新的な言語構成要素がいくつか用意されているため、ソフトウェア コンポーネントの開発が容易になります。</span><span class="sxs-lookup"><span data-stu-id="123d2-120">C# makes it easy to develop software components through several innovative language constructs, including:</span></span>

- <span data-ttu-id="123d2-121">メソッドのシグネチャをカプセル化する "*デリゲート*"。タイプ セーフなイベント通知を実行できます。</span><span class="sxs-lookup"><span data-stu-id="123d2-121">Encapsulated method signatures called *delegates*, which enable type-safe event notifications.</span></span>
- <span data-ttu-id="123d2-122">プロパティ。プライベート メンバー変数へのアクセサーとして機能します。</span><span class="sxs-lookup"><span data-stu-id="123d2-122">Properties, which serve as accessors for private member variables.</span></span>
- <span data-ttu-id="123d2-123">属性。実行時に型に関する宣言のメタデータを提供します。</span><span class="sxs-lookup"><span data-stu-id="123d2-123">Attributes, which provide declarative metadata about types at run time.</span></span>
- <span data-ttu-id="123d2-124">インライン XML ドキュメントのコメント。</span><span class="sxs-lookup"><span data-stu-id="123d2-124">Inline XML documentation comments.</span></span>
- <span data-ttu-id="123d2-125">統合言語クエリ (LINQ)。さまざまな種類のデータ ソースを対象とする組み込みのクエリ機能を提供します。</span><span class="sxs-lookup"><span data-stu-id="123d2-125">Language-Integrated Query (LINQ), which provides built-in query capabilities across different kinds of data sources.</span></span>
- <span data-ttu-id="123d2-126">パターンマッチング。データ型と値を調べることによって制御フローを有効にします。</span><span class="sxs-lookup"><span data-stu-id="123d2-126">Pattern matching, which enables control flow by inspecting data types and values.</span></span>

<span data-ttu-id="123d2-127">ネイティブ コンポーネントと対話するには、"相互運用" というプロセスを使用します。</span><span class="sxs-lookup"><span data-stu-id="123d2-127">You interact with native components through a process called "Interop".</span></span> <span data-ttu-id="123d2-128">C# プログラムで相互運用機能を使用すると、ネイティブの C++ アプリケーションで実行できる機能をほぼすべて実行できます。</span><span class="sxs-lookup"><span data-stu-id="123d2-128">Interop enables C# programs to do almost anything that a native C++ application can do.</span></span> <span data-ttu-id="123d2-129">さらに、ダイレクト メモリ アクセスがクリティカルの場合でも、ポインターと "unsafe" コードの概念がサポートされます。</span><span class="sxs-lookup"><span data-stu-id="123d2-129">C# even supports pointers and the concept of "unsafe" code for those cases in which direct memory access is critical.</span></span>

<span data-ttu-id="123d2-130">C# のビルド処理は、C や C++ よりも単純で Java よりも柔軟です。</span><span class="sxs-lookup"><span data-stu-id="123d2-130">The C# build process is simple compared to C and C++ and more flexible than in Java.</span></span> <span data-ttu-id="123d2-131">ヘッダー ファイルは分かれていません。また、メソッドや型を特定の順序で宣言する必要はありません。</span><span class="sxs-lookup"><span data-stu-id="123d2-131">There are no separate header files, and no requirement that methods and types be declared in a particular order.</span></span> <span data-ttu-id="123d2-132">C# のソース ファイルでは、クラス、構造体、インターフェイス、およびイベントをいくつでも定義できます。</span><span class="sxs-lookup"><span data-stu-id="123d2-132">A C# source file may define any number of classes, structs, interfaces, and events.</span></span>

<span data-ttu-id="123d2-133">その他に、C# の参照ドキュメントを紹介します。</span><span class="sxs-lookup"><span data-stu-id="123d2-133">The following are additional C# resources:</span></span>

## <a name="net-platform-architecture"></a><span data-ttu-id="123d2-137">.NET プラットフォーム アーキテクチャ</span><span class="sxs-lookup"><span data-stu-id="123d2-137">.NET Platform Architecture</span></span>

<span data-ttu-id="123d2-138">C# プログラムは、.NET で実行されます。これは、共通言語ランタイム (CLR) と呼ばれる仮想実行システムであり、クラス ライブラリの統合されたセットです。</span><span class="sxs-lookup"><span data-stu-id="123d2-138">C# programs run on .NET, a virtual execution system called the common language runtime (CLR) and a unified set of class libraries.</span></span> <span data-ttu-id="123d2-139">CLR は、国際規格である共通言語基盤 (CLI) の Microsoft による商用実装です。</span><span class="sxs-lookup"><span data-stu-id="123d2-139">The CLR is the commercial implementation by Microsoft of the common language infrastructure (CLI), an international standard.</span></span> <span data-ttu-id="123d2-140">CLI は、言語やライブラリをシームレスに連携する実行および開発環境を構築するための基盤です。</span><span class="sxs-lookup"><span data-stu-id="123d2-140">The CLI is the basis for creating execution and development environments in which languages and libraries work together seamlessly.</span></span>

<span data-ttu-id="123d2-141">C# で記述したソース コードは、CLI 仕様に準拠する[中間言語 (IL)](../../standard/managed-code.md) にコンパイルされます。</span><span class="sxs-lookup"><span data-stu-id="123d2-141">Source code written in C# is compiled into an [intermediate language (IL)](../../standard/managed-code.md) that conforms to the CLI specification.</span></span> <span data-ttu-id="123d2-142">IL コードおよびリソース (ビットマップや文字列など) は、アセンブリに保存されます。拡張子は一般的には .dll です。</span><span class="sxs-lookup"><span data-stu-id="123d2-142">The IL code and resources, such as bitmaps and strings, are stored in an assembly, typically with an extension of .dll.</span></span> <span data-ttu-id="123d2-143">アセンブリに含まれるマニフェストには、アセンブリの種類、バージョン、およびカルチャに関する情報が規定されています。</span><span class="sxs-lookup"><span data-stu-id="123d2-143">An assembly contains a manifest that provides information about the assembly's types, version, and culture.</span></span>

<span data-ttu-id="123d2-144">C# プログラムを実行すると、アセンブリが CLR に読み込まれます。</span><span class="sxs-lookup"><span data-stu-id="123d2-144">When the C# program is executed, the assembly is loaded into the CLR.</span></span> <span data-ttu-id="123d2-145">CLR によって Just-In-Time (JIT) コンパイルが実行され、IL コードはネイティブのマシン語命令に変換されます。</span><span class="sxs-lookup"><span data-stu-id="123d2-145">The CLR performs Just-In-Time (JIT) compilation to convert the IL code to native machine instructions.</span></span> <span data-ttu-id="123d2-146">CLR には、自動的なガベージ コレクション、例外処理、およびリソース管理に関する他のサービスが用意されています。</span><span class="sxs-lookup"><span data-stu-id="123d2-146">The CLR provides other services related to automatic garbage collection, exception handling, and resource management.</span></span> <span data-ttu-id="123d2-147">CLR で実行されるコードは、"マネージド コード" と呼ばれることがあります。反対に、特定のシステムを対象にしたネイティブのマシン語にコンパイルされたコードは、"アンマネージド コード" と呼ばれることがあります。</span><span class="sxs-lookup"><span data-stu-id="123d2-147">Code that's executed by the CLR is sometimes referred to as "managed code", in contrast to "unmanaged code", which is compiled into native machine language that targets a specific system.</span></span>

<span data-ttu-id="123d2-148">言語の相互運用性は、.NET の主要機能です。</span><span class="sxs-lookup"><span data-stu-id="123d2-148">Language interoperability is a key feature of .NET.</span></span> <span data-ttu-id="123d2-149">C# コンパイラによって生成された IL コードは、共通型の仕様 (CTS) に準拠しています。</span><span class="sxs-lookup"><span data-stu-id="123d2-149">IL code produced by the C# compiler conforms to the Common Type Specification (CTS).</span></span> <span data-ttu-id="123d2-150">C# から生成された IL コードは、F#、Visual Basic、C++ の .NET バージョンの他、20 を超える CTS 準拠言語で生成されたコードと相互運用性があります。</span><span class="sxs-lookup"><span data-stu-id="123d2-150">IL code generated from C# can interact with code that was generated from the .NET versions of F#, Visual Basic, C++, or any of more than 20 other CTS-compliant languages.</span></span> <span data-ttu-id="123d2-151">1 つのアセンブリには、異なる .NET 言語で記述されたモジュールを複数含めることができます。また、同じ言語で記述されている場合と同様に、型を参照することもできます。</span><span class="sxs-lookup"><span data-stu-id="123d2-151">A single assembly may contain multiple modules written in different .NET languages, and the types can reference each other as if they were written in the same language.</span></span>

<span data-ttu-id="123d2-152">.NET には、実行時のサービス以外にも、広範なライブラリが用意されています。</span><span class="sxs-lookup"><span data-stu-id="123d2-152">In addition to the run time services, .NET also includes extensive libraries.</span></span> <span data-ttu-id="123d2-153">これらのライブラリは、さまざまなワークロードをサポートします。</span><span class="sxs-lookup"><span data-stu-id="123d2-153">These libraries support many different workloads.</span></span> <span data-ttu-id="123d2-154">これらは、ファイルの入出力、XML 解析のための文字列操作、Web アプリケーションのフレームワーク、Windows フォーム コントロールなど、役に立つさまざまな機能を備えた名前空間に構成されています。</span><span class="sxs-lookup"><span data-stu-id="123d2-154">They are organized into namespaces that provide a wide variety of useful functionality for everything from file input and output to string manipulation to XML parsing, to web application frameworks to Windows Forms controls.</span></span> <span data-ttu-id="123d2-155">C# アプリケーションでは、一般に、.NET クラス ライブラリを広範囲に使用して、一般的な "配管工事" のような作業を処理しています。</span><span class="sxs-lookup"><span data-stu-id="123d2-155">The typical C# application uses the .NET class library extensively to handle common "plumbing" chores.</span></span>
