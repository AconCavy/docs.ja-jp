# <a name="common-patterns-for-delegates"></a><span data-ttu-id="38471-103">デリゲートの一般的なパターン</span><span class="sxs-lookup"><span data-stu-id="38471-103">Common Patterns for Delegates</span></span>

<span data-ttu-id="38471-105">デリゲートは、コンポーネント間の結合度を最小限にしたソフトウェア設計を可能にするメカニズムです。</span><span class="sxs-lookup"><span data-stu-id="38471-105">Delegates provide a mechanism that enables software designs involving minimal coupling between components.</span></span>

<span data-ttu-id="38471-106">この種の設計の代表的な例が LINQ です。</span><span class="sxs-lookup"><span data-stu-id="38471-106">One excellent example for this kind of design is LINQ.</span></span> <span data-ttu-id="38471-107">LINQ のクエリ式パターンは、そのすべての機能がデリゲートによって支えられています。</span><span class="sxs-lookup"><span data-stu-id="38471-107">The LINQ Query Expression Pattern relies on delegates for all of its features.</span></span> <span data-ttu-id="38471-108">簡単な例を考えてみましょう。</span><span class="sxs-lookup"><span data-stu-id="38471-108">Consider this simple example:</span></span>

```csharp
var smallNumbers = numbers.Where(n => n < 10);
```

<span data-ttu-id="38471-109">このコードは、一連の数値から値が 10 未満である数値のみを抽出するものです。</span><span class="sxs-lookup"><span data-stu-id="38471-109">This filters the sequence of numbers to only those less than the value 10.</span></span>
<span data-ttu-id="38471-110">`Where` メソッドは、デリゲートを使って、どの要素を抽出するかを判断しています。</span><span class="sxs-lookup"><span data-stu-id="38471-110">The `Where` method uses a delegate that determines which elements of a sequence pass the filter.</span></span> <span data-ttu-id="38471-111">LINQ クエリを作成するとき、その具体的な目的に合ったデリゲートの機能は、皆さんが実装することになります。</span><span class="sxs-lookup"><span data-stu-id="38471-111">When you create a LINQ query, you supply the implementation of the delegate for this specific purpose.</span></span>

<span data-ttu-id="38471-112">Where メソッドのプロトタイプは次のとおりです。</span><span class="sxs-lookup"><span data-stu-id="38471-112">The prototype for the Where method is:</span></span>

```csharp

```

<span data-ttu-id="38471-113">この例は、LINQ のすべてのメソッドに当てはまります。</span><span class="sxs-lookup"><span data-stu-id="38471-113">This example is repeated with all the methods that are part of LINQ.</span></span> <span data-ttu-id="38471-114">具体的なクエリを扱うコードには、すべてデリゲートが使われているのです。</span><span class="sxs-lookup"><span data-stu-id="38471-114">They all rely on delegates for the code that manages the specific query.</span></span> <span data-ttu-id="38471-115">きわめて強力な API デザイン パターンなので、しっかり覚えて自分のものにしてください。</span><span class="sxs-lookup"><span data-stu-id="38471-115">This API design pattern is a very powerful one to learn and understand.</span></span>

<span data-ttu-id="38471-116">この単純な例から、コンポーネント間の結合関係がデリゲートにはほとんど必要ないことがわかります。</span><span class="sxs-lookup"><span data-stu-id="38471-116">This simple example illustrates how delegates require very little coupling between components.</span></span> <span data-ttu-id="38471-117">特定の基本クラスから派生したクラスを作成する必要がないのです。</span><span class="sxs-lookup"><span data-stu-id="38471-117">You don't need to create a class that derives from a particular base class.</span></span> <span data-ttu-id="38471-118">特定のインターフェイスを実装する必要もありません。</span><span class="sxs-lookup"><span data-stu-id="38471-118">You don't need to implement a specific interface.</span></span>
