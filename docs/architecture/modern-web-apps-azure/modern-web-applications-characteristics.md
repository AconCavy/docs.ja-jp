<span data-ttu-id="9a29c-168">サーバー側の Blazor は、2019 年に ASP.NET Core 3.0 と共にリリースされました。</span><span class="sxs-lookup"><span data-stu-id="9a29c-168">Server-side Blazor was released in 2019 with ASP.NET Core 3.0.</span></span> <span data-ttu-id="9a29c-169">名前が示すように、それはサーバー上で実行され、クライアント ドキュメントへの変更をネットワーク経由でブラウザーにレンダリングします。</span><span class="sxs-lookup"><span data-stu-id="9a29c-169">As its name implies, it runs on the server, rendering changes to the client document back to the browser over the network.</span></span> <span data-ttu-id="9a29c-170">サーバー側の Blazor では、クライアント側の JavaScript を必要とせず、クライアント ページの操作ごとに個別のページ読み込みを必要としない、豊富なクライアント エクスペリエンスが提供されます。</span><span class="sxs-lookup"><span data-stu-id="9a29c-170">Server-side Blazor provides a rich client experience without requiring client-side JavaScript and without requiring separate page loads for each client page interaction.</span></span> <span data-ttu-id="9a29c-171">読み込まれたページの変更は、サーバーによって要求および処理された後、SignalR を使用してクライアントに返されます。</span><span class="sxs-lookup"><span data-stu-id="9a29c-171">Changes in the loaded page are requested from and processed by the server and then sent back to the client using SignalR.</span></span>

<span data-ttu-id="9a29c-172">クライアント側の Blazor は 2020 年にリリースされる予定であり、サーバー上で変更をレンダリングする必要がなくなります。</span><span class="sxs-lookup"><span data-stu-id="9a29c-172">Client-side Blazor will be released in 2020 and will eliminate the need to render changes on the server.</span></span> <span data-ttu-id="9a29c-173">代わりに、クライアント内で WebAssembly を活用して .NET コードが実行されます。</span><span class="sxs-lookup"><span data-stu-id="9a29c-173">Instead, it will leverage WebAssembly to run .NET code within the client.</span></span> <span data-ttu-id="9a29c-174">クライアントでは、必要に応じてサーバーに API 呼び出しを行ってデータを要求できますが、すべてのクライアント側の動作は WebAssembly 経由でクライアント内で実行されます。これはすべての主要ブラウザーで既にサポートされている Javascript ライブラリです。</span><span class="sxs-lookup"><span data-stu-id="9a29c-174">The client can still make API calls to the server if needed to request data, but all client-side behavior runs in the client via WebAssembly, which is already supported by all major browsers and is just a Javascript library.</span></span>
