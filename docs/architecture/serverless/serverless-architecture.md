<span data-ttu-id="c6a99-170">サーバーレスによってサーバー側の依存関係が抽象化され、開発者はビジネス ロジックに集中できます。</span><span class="sxs-lookup"><span data-stu-id="c6a99-170">Serverless abstracts the server-side dependencies and enables the developer to focus on business logic.</span></span> <span data-ttu-id="c6a99-171">たとえば、JavaScript フレームワークを使用してアプリを構築するモバイル開発者は、JavaScript でもサーバーレス関数を構築することができます。</span><span class="sxs-lookup"><span data-stu-id="c6a99-171">For example, a mobile developer who builds apps using a JavaScript framework can build serverless functions with JavaScript as well.</span></span> <span data-ttu-id="c6a99-172">サーバーレス ホストではオペレーティング システム、Node.js インスタンスを管理し、コードのホストや依存関係のパッケージ化などを行います。</span><span class="sxs-lookup"><span data-stu-id="c6a99-172">The serverless host manages the operating system, a Node.js instance to host the code, package dependencies, and more.</span></span> <span data-ttu-id="c6a99-173">開発者には、シンプルな入力セットと、出力用の標準テンプレートが提供されます。</span><span class="sxs-lookup"><span data-stu-id="c6a99-173">The developer is provided a simple set of inputs and a standard template for outputs.</span></span> <span data-ttu-id="c6a99-174">その後、ビジネス ロジックの構築とテストに専念できます。</span><span class="sxs-lookup"><span data-stu-id="c6a99-174">They then can focus on building and testing the business logic.</span></span> <span data-ttu-id="c6a99-175">そのため、既存のスキルを使用して、新しいプラットフォームについて学習したり、"サーバー側の開発者" になったりすることなく、モバイル アプリのバックエンド ロジックを構築することができます。</span><span class="sxs-lookup"><span data-stu-id="c6a99-175">They're therefore able to use existing skills to build the back-end logic for the mobile app without having to learn new platforms or become a "server-side developer."</span></span>

<span data-ttu-id="c6a99-177">ほとんどのクラウド プロバイダーは、モバイル開発ライフサイクル全体を簡略化する、モバイルベースのサーバーレス製品を提供しています。</span><span class="sxs-lookup"><span data-stu-id="c6a99-177">Most cloud providers offer mobile-based serverless products that simplify the entire mobile development lifecycle.</span></span> <span data-ttu-id="c6a99-178">製品では、データを保持するためのデータベースのプロビジョニングを自動化し、DevOps に関する懸念事項を処理し、クラウドベースのビルドとテスト フレームワークを提供し、開発者の優先言語を使用してビジネス プロセスをスクリプト化できる場合があります。</span><span class="sxs-lookup"><span data-stu-id="c6a99-178">The products may automate the provisioning of databases to persist data, handle DevOps concerns, provide cloud-based builds and testing frameworks and the ability to script business processes using the developer's preferred language.</span></span> <span data-ttu-id="c6a99-179">モバイル中心のサーバーレス アプローチに従うことで、プロセスを効率化できます。</span><span class="sxs-lookup"><span data-stu-id="c6a99-179">Following a mobile-centric serverless approach can streamline the process.</span></span> <span data-ttu-id="c6a99-180">サーバーレスでは、モバイル バックエンド用のサーバーのプロビジョニング、構成、および保守の大きなオーバーヘッドを取り除くことができます。</span><span class="sxs-lookup"><span data-stu-id="c6a99-180">Serverless removes the tremendous overhead of provisioning, configuring, and maintaining servers for the mobile back end.</span></span>

## <a name="internet-of-things-iot"></a><span data-ttu-id="c6a99-181">モノのインターネット (IoT)</span><span class="sxs-lookup"><span data-stu-id="c6a99-181">Internet of Things (IoT)</span></span>

<span data-ttu-id="c6a99-182">IoT は、一緒にネットワーク接続された物理オブジェクトを指します。</span><span class="sxs-lookup"><span data-stu-id="c6a99-182">IoT refers to physical objects that are networked together.</span></span> <span data-ttu-id="c6a99-183">"接続されたデバイス" または "スマート デバイス" と呼ばれることもあります。</span><span class="sxs-lookup"><span data-stu-id="c6a99-183">They're sometimes referred to as "connected devices" or "smart devices."</span></span> <span data-ttu-id="c6a99-184">自動車や自動販売機のすべてが接続され、在庫から温度や湿度などのセンサー データに至る情報が送信される場合があります。</span><span class="sxs-lookup"><span data-stu-id="c6a99-184">Everything from cars and vending machines may be connected and send information ranging from inventory to sensor data such as temperature and humidity.</span></span> <span data-ttu-id="c6a99-185">企業では、IoT により、監視と自動化を通じてビジネス プロセスを改善できます。</span><span class="sxs-lookup"><span data-stu-id="c6a99-185">In the enterprise, IoT provides business process improvements through monitoring and automation.</span></span> <span data-ttu-id="c6a99-186">IoT データを使用して、広い倉庫内の気候を調節したり、サプライ チェーンを通じて在庫を追跡したりする場合があります。</span><span class="sxs-lookup"><span data-stu-id="c6a99-186">IoT data may be used to regulate the climate in a large warehouse or track inventory through the supply chain.</span></span> <span data-ttu-id="c6a99-187">IoT では化学物質流出を感知し、煙が検知されたときに消防署に電話することができます。</span><span class="sxs-lookup"><span data-stu-id="c6a99-187">IoT can sense chemical spills and call the fire department when smoke is detected.</span></span>

<span data-ttu-id="c6a99-188">多くの場合、莫大な量のデバイスと情報により、メッセージをルーティングして処理するためのイベント ドリブン アーキテクチャが決まります。</span><span class="sxs-lookup"><span data-stu-id="c6a99-188">The sheer volume of devices and information often dictates an event-driven architecture to route and process messages.</span></span> <span data-ttu-id="c6a99-189">サーバーレスは、次のようないくつかの理由で理想的なソリューションです。</span><span class="sxs-lookup"><span data-stu-id="c6a99-189">Serverless is an ideal solution for several reasons:</span></span>

- <span data-ttu-id="c6a99-190">デバイスとデータの量の増加に伴い、スケーリングすることができる。</span><span class="sxs-lookup"><span data-stu-id="c6a99-190">Enables scale as the volume of devices and data increases.</span></span>
- <span data-ttu-id="c6a99-191">新しいデバイスとセンサーをサポートするための新しいエンドポイントの追加に対応する。</span><span class="sxs-lookup"><span data-stu-id="c6a99-191">Accommodates adding new endpoints to support new devices and sensors.</span></span>
- <span data-ttu-id="c6a99-192">独立したバージョン管理を容易にし、開発者がシステム全体をデプロイすることなく、特定のデバイスのビジネス ロジックを更新できるようにする。</span><span class="sxs-lookup"><span data-stu-id="c6a99-192">Facilitates independent versioning so developers can update the business logic for a specific device without having to deploy the entire system.</span></span>
- <span data-ttu-id="c6a99-193">回復性、およびダウンタイムの削減。</span><span class="sxs-lookup"><span data-stu-id="c6a99-193">Resiliency and less downtime.</span></span>

<span data-ttu-id="c6a99-194">IoT の広汎性により、[Azure IoT Hub](https://docs.microsoft.com/azure/iot-hub)などの IoT の懸念事項に特に焦点を当てる、複数のサーバーレス製品が生成されました。</span><span class="sxs-lookup"><span data-stu-id="c6a99-194">The pervasiveness of IoT has resulted in several serverless products that focus specifically on IoT concerns, such as [Azure IoT Hub](https://docs.microsoft.com/azure/iot-hub).</span></span> <span data-ttu-id="c6a99-195">サーバーレスでは、デバイスの登録、ポリシーの適用、追跡、さらには*エッジ* のデバイスへのコードのデプロイなどのタスクを自動化します。</span><span class="sxs-lookup"><span data-stu-id="c6a99-195">Serverless automates tasks such as device registration, policy enforcement, tracking, and even deployment of code to devices at *the edge*.</span></span> <span data-ttu-id="c6a99-196">エッジは、インターネット (ただし、アクティブな部分ではない) に接続されている、センサーやアクチュエータなどのデバイスを指します。</span><span class="sxs-lookup"><span data-stu-id="c6a99-196">The edge refers to devices like sensors and actuators that are connected to, but not an active part of, the Internet.</span></span>
