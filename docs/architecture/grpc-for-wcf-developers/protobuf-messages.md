# <a name="protobuf-messages"></a><span data-ttu-id="e8370-103">Protobuf メッセージ</span><span class="sxs-lookup"><span data-stu-id="e8370-103">Protobuf messages</span></span>

<span data-ttu-id="e8370-104">このセクションでは `.proto` ファイルで Protobuf メッセージを宣言する方法、フィールド番号と型の基本的な概念について説明C#し、`protoc` コンパイラによって生成されるコードを確認する方法について説明します。</span><span class="sxs-lookup"><span data-stu-id="e8370-104">This section covers how to declare Protobuf messages in `.proto` files, explains the fundamental concepts of field numbers and types, and looks at the C# code that is generated by the `protoc` compiler.</span></span> <span data-ttu-id="e8370-105">この章の残りの部分では、Protobuf でさまざまな種類のデータがどのように表現されるかについて詳しく説明します。</span><span class="sxs-lookup"><span data-stu-id="e8370-105">The rest of the chapter will look in more detail at how different types of data are represented in Protobuf.</span></span>

## <a name="declaring-a-message"></a><span data-ttu-id="e8370-106">メッセージの宣言</span><span class="sxs-lookup"><span data-stu-id="e8370-106">Declaring a message</span></span>

<span data-ttu-id="e8370-107">WCF では、次の例のように、株式市場取引アプリケーションの `Stock` クラスを定義することができます。</span><span class="sxs-lookup"><span data-stu-id="e8370-107">In WCF, a `Stock` class for a stock market trading application might be defined like the following example:</span></span>

```csharp
namespace TraderSys
{
    [DataContract]
    public class Stock
    {
        [DataMember]
        public int Id { get; set;}
        [DataMember]
        public string Symbol { get; set;}
        [DataMember]
        public string DisplayName { get; set;}
        [DataMember]
        public int MarketId { get; set; }
    }
}
```

<span data-ttu-id="e8370-108">Protobuf で同等のクラスを実装するには、そのクラスを `.proto` ファイルで宣言する必要があります。</span><span class="sxs-lookup"><span data-stu-id="e8370-108">To implement the equivalent class in Protobuf, it must be declared in the `.proto` file.</span></span> <span data-ttu-id="e8370-109">その後、`protoc` コンパイラによって、ビルド処理の一部として .NET クラスが生成されます。</span><span class="sxs-lookup"><span data-stu-id="e8370-109">The `protoc` compiler will then generate the .NET class as part of the build process.</span></span>

```protobuf
syntax "proto3";

option csharp_namespace = "TraderSys";

message Stock {

    int32 id = 1;
    string symbol = 2;
    string display_name = 3;
    int32 market_id = 4;

}  
```

<span data-ttu-id="e8370-110">最初の行は、使用されている構文のバージョンを宣言します。</span><span class="sxs-lookup"><span data-stu-id="e8370-110">The first line declares the syntax version being used.</span></span> <span data-ttu-id="e8370-111">言語のバージョン3は2016でリリースされ、gRPC サービスに推奨されるバージョンです。</span><span class="sxs-lookup"><span data-stu-id="e8370-111">Version 3 of the language was released in 2016 and is the recommended version for gRPC services.</span></span>

<span data-ttu-id="e8370-112">`option csharp_namespace` 行では、生成C#された型に使用する名前空間を指定します。</span><span class="sxs-lookup"><span data-stu-id="e8370-112">The `option csharp_namespace` line specifies the namespace to be used for the generated C# types.</span></span> <span data-ttu-id="e8370-113">`.proto` ファイルが他の言語用にコンパイルされている場合、このオプションは無視されます。</span><span class="sxs-lookup"><span data-stu-id="e8370-113">This option will be ignored when the `.proto` file is compiled for other languages.</span></span> <span data-ttu-id="e8370-114">Protobuf ファイルには、複数の言語の言語固有のオプションが含まれているのが一般的です。</span><span class="sxs-lookup"><span data-stu-id="e8370-114">It is common for Protobuf files to contain language-specific options for several languages.</span></span>

<span data-ttu-id="e8370-115">`Stock` メッセージ定義では、それぞれ型、名前、およびフィールド番号を持つ4つのフィールドが指定されています。</span><span class="sxs-lookup"><span data-stu-id="e8370-115">The `Stock` message definition specifies four fields, each with a type, a name, and a field number.</span></span>

## <a name="field-numbers"></a><span data-ttu-id="e8370-116">フィールド番号</span><span class="sxs-lookup"><span data-stu-id="e8370-116">Field numbers</span></span>

<span data-ttu-id="e8370-117">フィールド番号は Protobuf の重要な部分です。</span><span class="sxs-lookup"><span data-stu-id="e8370-117">Field numbers are an important part of Protobuf.</span></span> <span data-ttu-id="e8370-118">これらは、バイナリエンコードデータ内のフィールドを識別するために使用されます。つまり、サービスのバージョンごとに変更することはできません。</span><span class="sxs-lookup"><span data-stu-id="e8370-118">They're used to identify fields in the binary encoded data, which means they can't change from version to version of your service.</span></span> <span data-ttu-id="e8370-119">長所は、下位互換性と上位互換性が可能であることです。</span><span class="sxs-lookup"><span data-stu-id="e8370-119">The advantage is that backward and forward compatibility is possible.</span></span> <span data-ttu-id="e8370-120">クライアントとサービスは、不明な値が処理される可能性がある限り、知られていないフィールド番号を無視します。</span><span class="sxs-lookup"><span data-stu-id="e8370-120">Clients and services will simply ignore field numbers they don't know about, as long as the possibility of missing values is handled.</span></span>

<span data-ttu-id="e8370-121">バイナリ形式では、フィールド番号が型識別子と結合されます。</span><span class="sxs-lookup"><span data-stu-id="e8370-121">In the binary format, the field number is combined with a type identifier.</span></span> <span data-ttu-id="e8370-122">1 ~ 15 のフィールド番号は、その型で1バイトとしてエンコードできます。16から2047の数値は2バイトになります。</span><span class="sxs-lookup"><span data-stu-id="e8370-122">Field numbers from 1 to 15 can be encoded with their type as a single byte; numbers from 16 to 2047 take 2 bytes.</span></span> <span data-ttu-id="e8370-123">何らかの理由でメッセージに2047を超えるフィールドが必要な場合は、より高い値にすることができます。</span><span class="sxs-lookup"><span data-stu-id="e8370-123">You can go higher if you need more than 2047 fields on a message for any reason.</span></span> <span data-ttu-id="e8370-124">フィールド番号が 1 ~ 15 の1バイト識別子ではパフォーマンスが向上します。そのため、最も基本的で頻繁に使用されるフィールドで使用する必要があります。</span><span class="sxs-lookup"><span data-stu-id="e8370-124">The single byte identifiers for field numbers 1 to 15 offer better performance, so you should use them for the most basic, frequently used fields.</span></span>

## <a name="the-generated-code"></a><span data-ttu-id="e8370-130">生成されたコード</span><span class="sxs-lookup"><span data-stu-id="e8370-130">The generated code</span></span>

<span data-ttu-id="e8370-131">アプリケーションをビルドすると、Protobuf はメッセージごとにクラスを作成し、ネイティブな型をC#型にマップします。</span><span class="sxs-lookup"><span data-stu-id="e8370-131">When you build your application, Protobuf creates classes for each of your messages, mapping its native types to C# types.</span></span> <span data-ttu-id="e8370-132">生成される `Stock` 型には、次のシグネチャがあります。</span><span class="sxs-lookup"><span data-stu-id="e8370-132">The generated `Stock` type would have the following signature:</span></span>

```csharp
public class Stock
{
    public int Id { get; set; }
    public string Symbol { get; set; }
    public string DisplayName { get; set; }
    public int MarketId { get; set; }
}
```

<span data-ttu-id="e8370-133">生成される実際のコードは、これよりはるかに複雑です。各クラスには、それ自体をバイナリワイヤ形式にシリアル化および逆シリアル化するために必要なすべてのコードが含まれているためです。</span><span class="sxs-lookup"><span data-stu-id="e8370-133">The actual code that is generated is far more complicated than this, because each class contains all the code necessary to serialize and deserialize itself to the binary wire format.</span></span>
