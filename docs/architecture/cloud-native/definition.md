## <a name="the-cloud"></a><span data-ttu-id="5dd70-143">クラウド.</span><span class="sxs-lookup"><span data-stu-id="5dd70-143">The cloud…</span></span>

<span data-ttu-id="5dd70-144">クラウド ネイティブ システムは、クラウド サービス モデルを最大限に活用します。</span><span class="sxs-lookup"><span data-stu-id="5dd70-144">Cloud-native systems take full advantage of the cloud service model.</span></span>

<span data-ttu-id="5dd70-145">動的な仮想化クラウド環境での繁栄を目的として設計されたこれらのシステムは、[プラットフォームをサービスとして(PaaS)](https://azure.microsoft.com/overview/what-is-paas/)コンピューティングインフラストラクチャとマネージドサービスを幅広く利用しています。</span><span class="sxs-lookup"><span data-stu-id="5dd70-145">Designed to thrive in a dynamic, virtualized cloud environment, these systems make extensive use of [Platform as a Service (PaaS)](https://azure.microsoft.com/overview/what-is-paas/) compute infrastructure and managed services.</span></span> <span data-ttu-id="5dd70-146">基盤となるインフラストラクチャは、数分でプロビジョニングされ、オートメーションによってサイズ変更、拡大縮小、移動、または破棄される *、使い捨て*として扱われます。</span><span class="sxs-lookup"><span data-stu-id="5dd70-146">They treat the underlying infrastructure as *disposable* - provisioned in minutes and resized, scaled, moved, or destroyed on demand – via automation.</span></span>

<span data-ttu-id="5dd70-147">[ペット対牛](https://medium.com/@Joachim8675309/devops-concepts-pets-vs-cattle-2380b5aab313)の広く受け入れられているDevOpsの概念を考えてみましょう。</span><span class="sxs-lookup"><span data-stu-id="5dd70-147">Consider the widely accepted DevOps concept of [Pets vs. Cattle](https://medium.com/@Joachim8675309/devops-concepts-pets-vs-cattle-2380b5aab313).</span></span> <span data-ttu-id="5dd70-148">従来のデータセンターでは、サーバーは*Pets*として扱われます: 物理マシン、意味のある名前を与えられ、世話をします。</span><span class="sxs-lookup"><span data-stu-id="5dd70-148">In a traditional data center, servers are treated as *Pets*: a physical machine, given a meaningful name, and cared for.</span></span> <span data-ttu-id="5dd70-149">同じマシンにリソースを追加してスケールアップします (スケールアップ)。</span><span class="sxs-lookup"><span data-stu-id="5dd70-149">You scale by adding more resources to the same machine (scaling up).</span></span> <span data-ttu-id="5dd70-150">サーバーが病気になった場合は、健康に戻って看護します。</span><span class="sxs-lookup"><span data-stu-id="5dd70-150">If the server becomes sick, you nurse it back to health.</span></span> <span data-ttu-id="5dd70-151">サーバーが使用できなくなった場合、全員が通知します。</span><span class="sxs-lookup"><span data-stu-id="5dd70-151">Should the server become unavailable, everyone notices.</span></span>

<span data-ttu-id="5dd70-152">*牛*のサービスモデルは異なります。</span><span class="sxs-lookup"><span data-stu-id="5dd70-152">The *Cattle* service model is different.</span></span> <span data-ttu-id="5dd70-153">各インスタンスを仮想マシンまたはコンテナーとしてプロビジョニングします。</span><span class="sxs-lookup"><span data-stu-id="5dd70-153">You provision each instance as a virtual machine or container.</span></span> <span data-ttu-id="5dd70-154">これらは同じで、Service-01、Service-02 などのシステム識別子が割り当てられています。</span><span class="sxs-lookup"><span data-stu-id="5dd70-154">They're identical and assigned a system identifier such as Service-01, Service-02, and so on.</span></span> <span data-ttu-id="5dd70-155">スケールするには、その他のサイズを作成します (スケール アウト)。</span><span class="sxs-lookup"><span data-stu-id="5dd70-155">You scale by creating more of them (scaling out).</span></span> <span data-ttu-id="5dd70-156">1つが利用できなくなったら、誰も気付かない。</span><span class="sxs-lookup"><span data-stu-id="5dd70-156">When one becomes unavailable, nobody notices.</span></span>

<span data-ttu-id="5dd70-157">牛モデルは*不変のインフラを*受け入れています。</span><span class="sxs-lookup"><span data-stu-id="5dd70-157">The cattle model embraces *immutable infrastructure*.</span></span> <span data-ttu-id="5dd70-158">サーバーは修復も変更もされません。</span><span class="sxs-lookup"><span data-stu-id="5dd70-158">Servers aren't repaired or modified.</span></span> <span data-ttu-id="5dd70-159">1 つが失敗した場合、または更新が必要な場合は、破棄され、新しいものがプロビジョニングされます。</span><span class="sxs-lookup"><span data-stu-id="5dd70-159">If one fails or requires updating, it's destroyed and a new one is provisioned – all done via automation.</span></span>

<span data-ttu-id="5dd70-160">クラウドネイティブシステムは、牛サービスモデルを採用しています。</span><span class="sxs-lookup"><span data-stu-id="5dd70-160">Cloud-native systems embrace the Cattle service model.</span></span> <span data-ttu-id="5dd70-161">インフラストラクチャは、実行しているマシンに関係なく、スケールインまたはスケールアウトに応じて実行し続けます。</span><span class="sxs-lookup"><span data-stu-id="5dd70-161">They continue to run as the infrastructure scales in or out with no regard to the machines upon which they're running.</span></span>

<span data-ttu-id="5dd70-162">Azure クラウド プラットフォームは、自動スケーリング、自己修復、および監視機能を備えた、この種の高弾性インフラストラクチャをサポートします。</span><span class="sxs-lookup"><span data-stu-id="5dd70-162">The Azure cloud platform supports this type of highly elastic infrastructure with automatic scaling, self-healing, and monitoring capabilities.</span></span>
