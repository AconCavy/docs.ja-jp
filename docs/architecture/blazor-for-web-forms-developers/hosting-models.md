## <a name="blazor-server-apps"></a><span data-ttu-id="384a5-133">ブレイザサーバーアプリ</span><span class="sxs-lookup"><span data-stu-id="384a5-133">Blazor Server apps</span></span>

<span data-ttu-id="384a5-134">[Blazor アーキテクチャ](architecture-comparison.md#blazor)の説明から、Blazor コンポーネントは、出力を`RenderTree`.</span><span class="sxs-lookup"><span data-stu-id="384a5-134">Recall from the [Blazor architecture](architecture-comparison.md#blazor) discussion that Blazor components render their output to an intermediate abstraction called a `RenderTree`.</span></span> <span data-ttu-id="384a5-135">次に、Blazor フレームワークは、レンダリングされたものと以前にレンダリングされたものを比較します。</span><span class="sxs-lookup"><span data-stu-id="384a5-135">The Blazor framework then compares what was rendered with what was previously rendered.</span></span> <span data-ttu-id="384a5-136">この違いは DOM に適用されます。</span><span class="sxs-lookup"><span data-stu-id="384a5-136">The differences are applied to the DOM.</span></span> <span data-ttu-id="384a5-137">Blazor コンポーネントは、レンダリングされた出力の適用方法から切り離されます。</span><span class="sxs-lookup"><span data-stu-id="384a5-137">Blazor components are decoupled from how their rendered output is applied.</span></span> <span data-ttu-id="384a5-138">したがって、コンポーネント自体は、UI を更新するプロセスと同じプロセスで実行する必要はありません。</span><span class="sxs-lookup"><span data-stu-id="384a5-138">Consequently, the components themselves don't have to run in the same process as the process updating the UI.</span></span> <span data-ttu-id="384a5-139">実際、彼らは同じマシンで実行する必要はありません。</span><span class="sxs-lookup"><span data-stu-id="384a5-139">In fact, they don't even have to run on the same machine.</span></span>
