 <span data-ttu-id="35e98-165">これらの変更を行うと、プロファイリングがシステム全体で有効になります。</span><span class="sxs-lookup"><span data-stu-id="35e98-165">Note that these changes will enable profiling on a system-wide basis.</span></span> <span data-ttu-id="35e98-166">以降に実行するすべてのマネージド アプリケーションがプロファイルされるのを避けるには、ターゲット コンピューターを再起動した後で、システム環境変数を削除します。</span><span class="sxs-lookup"><span data-stu-id="35e98-166">To prevent every managed application that subsequently runs from being profiled, you should delete the system environment variables after you restart the target computer.</span></span>  
  
 <span data-ttu-id="35e98-167">この方法では、すべての CLR プロセスもプロファイリングの対象になります。</span><span class="sxs-lookup"><span data-stu-id="35e98-167">This technique also leads to every CLR process getting profiled.</span></span> <span data-ttu-id="35e98-168">プロファイラーは、現在のプロセスが対象かどうかを検出するコールバックを初期化するロジックを[ICorProfilerCallback::初期化](icorprofilercallback-initialize-method.md)する必要があります。</span><span class="sxs-lookup"><span data-stu-id="35e98-168">The profiler should add logic to its [ICorProfilerCallback::Initialize](icorprofilercallback-initialize-method.md) callback to detect whether the current process is of interest.</span></span> <span data-ttu-id="35e98-169">プロファイリングの対象ではない場合、プロファイラーは、初期化を実行せずにコールバックからエラーを返すことができます。</span><span class="sxs-lookup"><span data-stu-id="35e98-169">If it is not, the profiler can fail the callback without performing the initialization.</span></span>
