 <span data-ttu-id="b3233-125">プラットフォーム呼び出しは、エクスポート関数を検索して、その引数を実行時にマーシャリングするために、メタデータに依存します。</span><span class="sxs-lookup"><span data-stu-id="b3233-125">Platform invoke relies on metadata to locate exported functions and marshal their arguments at run time.</span></span> <span data-ttu-id="b3233-126">次に、このプロセスの図を示します。</span><span class="sxs-lookup"><span data-stu-id="b3233-126">The following illustration shows this process.</span></span>  
  
 ![プラットフォーム呼び出しを示す図。](./media/consuming-unmanaged-dll-functions/platform-invoke-call.gif)  
  
 <span data-ttu-id="b3233-128">プラットフォーム呼び出しがアンマネージ関数を呼び出すと、以下の一連のアクションが実行されます。</span><span class="sxs-lookup"><span data-stu-id="b3233-128">When platform invoke calls an unmanaged function, it performs the following sequence of actions:</span></span>  
  
1. <span data-ttu-id="b3233-129">関数を含む DLL を検索します。</span><span class="sxs-lookup"><span data-stu-id="b3233-129">Locates the DLL containing the function.</span></span>

3. <span data-ttu-id="b3233-131">メモリ内の関数のアドレスを検索し、その引数をスタックにプッシュし、必要に応じてデータをマーシャリングします。</span><span class="sxs-lookup"><span data-stu-id="b3233-131">Locates the address of the function in memory and pushes its arguments onto the stack, marshaling data as required.</span></span>
