# <a name="net-core-and-open-source"></a><span data-ttu-id="1ba56-103">.NET Core とオープン ソース</span><span class="sxs-lookup"><span data-stu-id="1ba56-103">.NET Core and open source</span></span>

## <a name="what-is-net-core"></a><span data-ttu-id="1ba56-105">.NET Core とは何ですか?</span><span class="sxs-lookup"><span data-stu-id="1ba56-105">What is .NET Core?</span></span>  

<span data-ttu-id="1ba56-106">.NET Core は、モジュール式のクロスプラットフォームかつオープンソースを実装した汎用の .NET Standard です。</span><span class="sxs-lookup"><span data-stu-id="1ba56-106">.NET Core is a general purpose, modular, cross-platform, and open-source implementation of the .NET Standard.</span></span> <span data-ttu-id="1ba56-107">.NET Framework と同じ API の多くが含まれるほか (しかし、.NET Core の方が数が少ない)、ランタイム、フレームワーク、コンパイラ、さまざまなオペレーティング システムやチップ ターゲットをサポートするツールのコンポーネントが含まれます。</span><span class="sxs-lookup"><span data-stu-id="1ba56-107">It contains many of the same APIs as the .NET Framework (but .NET Core is a smaller set) and includes runtime, framework, compiler, and tools components that support a variety of operating systems and chip targets.</span></span> <span data-ttu-id="1ba56-108">.NET Core の実装は、主に ASP.NET Core のワークロードによるものですが、新しい実装の必要性とユーザーの要望にも後押しされました。</span><span class="sxs-lookup"><span data-stu-id="1ba56-108">The .NET Core implementation was primarily driven by the ASP.NET Core workloads but also by the need and desire to have a more modern implementation.</span></span> <span data-ttu-id="1ba56-109">これは、デバイス、クラウド、Embedded/IoT のシナリオで使用できます。</span><span class="sxs-lookup"><span data-stu-id="1ba56-109">It can be used in device, cloud, and embedded/IoT scenarios.</span></span>  
  
<span data-ttu-id="1ba56-110">.NET Core の使用を開始するには、.NET チュートリアル 「[Hello World in 10 minutes](https://dotnet.microsoft.com/learn/dotnet/hello-world-tutorial/intro)」 (10 分で Hello World) を参照してください。</span><span class="sxs-lookup"><span data-stu-id="1ba56-110">To get started with .NET Core, visit the .NET tutorial [Hello World in 10 minutes](https://dotnet.microsoft.com/learn/dotnet/hello-world-tutorial/intro).</span></span>  
  
<span data-ttu-id="1ba56-111">.NET Core の主な特徴を次に示します。</span><span class="sxs-lookup"><span data-stu-id="1ba56-111">The main characteristics of .NET Core are:</span></span>
  
- <span data-ttu-id="1ba56-112">**クロス プラットフォーム:** .NET Core には、ターゲットとするプラットフォームに関係なく、必要とするアプリケーション機能を実装したり、そのコードを再使用したりするための主な機能が備わっています。</span><span class="sxs-lookup"><span data-stu-id="1ba56-112">**Cross-platform:** .NET Core provides key functionality to implement the app features you need and reuse this code regardless of your platform target.</span></span> <span data-ttu-id="1ba56-113">現在、次の 3 つの主要オペレーティング システム (OS) がサポートされています:Windows、Linux、macOS の 3 つの主要オペレーティング システムをサポートしています。</span><span class="sxs-lookup"><span data-stu-id="1ba56-113">It currently supports three main operating systems (OS): Windows, Linux, and macOS.</span></span> <span data-ttu-id="1ba56-114">サポートされている複数の OS で、修正せずに動作するアプリやライブラリを作成することができます。</span><span class="sxs-lookup"><span data-stu-id="1ba56-114">You can write apps and libraries that run unmodified across supported operating systems.</span></span> <span data-ttu-id="1ba56-115">サポートされるオペレーティング システムの一覧については、[「.NET Core Roadmap」](https://github.com/dotnet/core/blob/master/roadmap.md) (.NET Core ロードマップ) を参照してください。</span><span class="sxs-lookup"><span data-stu-id="1ba56-115">To see the list of supported operating systems, visit [.NET Core roadmap](https://github.com/dotnet/core/blob/master/roadmap.md).</span></span>
  
- <span data-ttu-id="1ba56-116">**オープン ソース:** .NET Core は [.NET Foundation ](https://www.dotnetfoundation.org/)が管理している多くのプロジェクトの 1 つで、[GitHub](https://github.com/) で入手することができます。</span><span class="sxs-lookup"><span data-stu-id="1ba56-116">**Open source:** .NET Core is one of the many projects under the stewardship of the [.NET Foundation](https://www.dotnetfoundation.org/) and is available on [GitHub](https://github.com/).</span></span> <span data-ttu-id="1ba56-117">.NET Core をオープン ソース プロジェクトとして使用すると、開発プロセスの透明性が高まるほか、コミュニティが活発化して交流が促進されます。</span><span class="sxs-lookup"><span data-stu-id="1ba56-117">As an open-source project, .NET Core promotes a more transparent development process and an active and engaged community.</span></span>  
  
- <span data-ttu-id="1ba56-118">**柔軟な展開:** アプリを展開する主な方法には、フレームワークに依存する展開と自己完結型の展開の 2 つがあります。</span><span class="sxs-lookup"><span data-stu-id="1ba56-118">**Flexible deployment:** there are two main ways to deploy your app: framework-dependent deployment or self-contained deployment.</span></span> <span data-ttu-id="1ba56-119">フレームワークに依存して展開する場合、アプリとサード パーティの依存関係のみがインストールされ、アプリを使用できるようにするには、.NET Core のシステム全体のバージョンが必要です。</span><span class="sxs-lookup"><span data-stu-id="1ba56-119">With framework-dependent deployment, only your app and third-party dependencies are installed and your app depends on a system-wide version of .NET Core to be present.</span></span> <span data-ttu-id="1ba56-120">自己完結型で展開する場合、アプリの作成に使用される .NET Core バージョンが、アプリやサード パーティの依存関係と共に展開され、他のバージョンと並行して実行することができます。</span><span class="sxs-lookup"><span data-stu-id="1ba56-120">With self-contained deployment, the .NET Core version used to build your application is also deployed along with your app and third-party dependencies and can run side by side with other versions.</span></span> <span data-ttu-id="1ba56-121">詳しくは、「[.NET Core アプリケーション展開](../../core/deploying/index.md)」をご覧ください。</span><span class="sxs-lookup"><span data-stu-id="1ba56-121">For more information, see [.NET Core Application Deployment](../../core/deploying/index.md).</span></span>

- <span data-ttu-id="1ba56-122">**モジュール形式:** .NET Core は、小規模のアセンブリ パッケージで NuGet を介してリリースされるためモジュール形式となっています。</span><span class="sxs-lookup"><span data-stu-id="1ba56-122">**Modular:** .NET Core is modular because it's released through NuGet in smaller assembly packages.</span></span> <span data-ttu-id="1ba56-123">.NET Core はコア機能のほとんどが含まれる 1 つの大きなアセンブリではなく、中心的な機能が含まれるより小さなパッケージとして利用できます。</span><span class="sxs-lookup"><span data-stu-id="1ba56-123">Rather than one large assembly that contains most of the core functionality, .NET Core is made available as smaller, feature-centric packages.</span></span> <span data-ttu-id="1ba56-124">このモジュール性によって、開発モデルがよりアジャイル化されるため、必要な NuGet パッケージだけが含まれるようにアプリを最適化することができます。</span><span class="sxs-lookup"><span data-stu-id="1ba56-124">This modularity enables a more agile development model for us and allows you to optimize your app to include just the NuGet packages you need.</span></span> <span data-ttu-id="1ba56-125">小さいアプリ領域の利点には、セキュリティの強化、サービスの削減、パフォーマンスの向上、従量課金モデルによるコスト削減などがあります。</span><span class="sxs-lookup"><span data-stu-id="1ba56-125">The benefits of a smaller app surface area include tighter security, reduced servicing, improved performance, and decreased costs in a pay-for-what-you-use model.</span></span>
