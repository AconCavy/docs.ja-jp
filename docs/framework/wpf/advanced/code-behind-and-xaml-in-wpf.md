- <span data-ttu-id="0fabf-120">マークアップ コンパイル ビルド アクションの既定の動作では、分離コード側の部分クラス定義で派生を空白のままにできます。</span><span class="sxs-lookup"><span data-stu-id="0fabf-120">Note that under the default behavior of the markup compile build actions, you can leave the derivation blank in the partial class definition on the code-behind side.</span></span> <span data-ttu-id="0fabf-121">コンパイルされた結果は、ページ ルートのバッキング型が、指定されていない場合でも部分クラスの基礎であると想定します。</span><span class="sxs-lookup"><span data-stu-id="0fabf-121">The compiled result will assume the page root's backing type to be the basis for the partial class, even if it not specified.</span></span> <span data-ttu-id="0fabf-122">ただし、この動作に依存することはベスト プラクティスではありません。</span><span class="sxs-lookup"><span data-stu-id="0fabf-122">However, relying on this behavior is not a best practice.</span></span>  
  
- <span data-ttu-id="0fabf-123">分離コードで記述するイベント ハンドラーは、インスタンス メソッドである必要があり、静的メソッドで指定することはできません。</span><span class="sxs-lookup"><span data-stu-id="0fabf-123">The event handlers you write in the code-behind must be instance methods and cannot be static methods.</span></span> <span data-ttu-id="0fabf-124">これらのメソッドは、 で識別される CLR 名前空間内の部分`x:Class`クラスで定義する必要があります。</span><span class="sxs-lookup"><span data-stu-id="0fabf-124">These methods must be defined by the partial class within the CLR namespace identified by `x:Class`.</span></span> <span data-ttu-id="0fabf-125">イベント ハンドラの名前を修飾して、プロセッサに対[!INCLUDE[TLA2#tla_xaml](../../../../includes/tla2sharptla-xaml-md.md)]して、別のクラス スコープでのイベント ワイヤリングのイベント ハンドラを探す指示を行うことはできません。</span><span class="sxs-lookup"><span data-stu-id="0fabf-125">You cannot qualify the name of an event handler to instruct a [!INCLUDE[TLA2#tla_xaml](../../../../includes/tla2sharptla-xaml-md.md)] processor to look for an event handler for event wiring in a different class scope.</span></span>
