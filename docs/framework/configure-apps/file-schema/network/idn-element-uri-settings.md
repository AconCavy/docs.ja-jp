## <a name="remarks"></a><span data-ttu-id="49df8-121">コメント</span><span class="sxs-lookup"><span data-stu-id="49df8-121">Remarks</span></span>

<span data-ttu-id="49df8-126">IRI のサポートを有効にするには、次の2つの変更が必要です。</span><span class="sxs-lookup"><span data-stu-id="49df8-126">To enable support for IRI, the following two changes are required:</span></span>

    ```xml  
    <section name="uri" type="System.Configuration.UriSection, System, Version=2.0.0.0, Culture=neutral, PublicKeyToken=b77a5c561934e089" />  
    ```  
  
2. <span data-ttu-id="49df8-128">国際化ドメイン名 (IDN) の解析をドメイン名に適用するかどうか、および IRI 解析規則を適用するかどうかを指定します。</span><span class="sxs-lookup"><span data-stu-id="49df8-128">Specify whether you want Internationalized Domain Name (IDN) parsing applied to the domain name and whether IRI parsing rules should be applied.</span></span> <span data-ttu-id="49df8-129">これは、machine.config ファイルまたは app.config ファイルで指定できます。</span><span class="sxs-lookup"><span data-stu-id="49df8-129">This can be done in the machine.config or in the app.config file.</span></span>

 <span data-ttu-id="49df8-130">IDN には、使用する DNS サーバーに応じて、次の3つの値を指定できます。</span><span class="sxs-lookup"><span data-stu-id="49df8-130">There are three possible values for IDN depending on the DNS servers that are used:</span></span>

- <span data-ttu-id="49df8-131">idn enabled = すべて</span><span class="sxs-lookup"><span data-stu-id="49df8-131">idn enabled = All</span></span>  

     <span data-ttu-id="49df8-132">この値により、すべての Unicode ドメイン名が Punycode に相当する (IDN 名) に変換されます。</span><span class="sxs-lookup"><span data-stu-id="49df8-132">This value will convert any Unicode domain names to their Punycode equivalents (IDN names).</span></span>

- <span data-ttu-id="49df8-133">idn enabled = AllExceptIntranet</span><span class="sxs-lookup"><span data-stu-id="49df8-133">idn enabled = AllExceptIntranet</span></span>

     <span data-ttu-id="49df8-134">この値を指定すると、ローカルイントラネット上にないすべての Unicode ドメイン名が、Punycode に相当する (IDN 名) を使用するように変換されます。</span><span class="sxs-lookup"><span data-stu-id="49df8-134">This value will convert all Unicode domain names not on the local Intranet to use the Punycode equivalents (IDN names).</span></span> <span data-ttu-id="49df8-135">この場合は、イントラネットで使用される DNS サーバーが Unicode 名前解決をサポートする必要があります。</span><span class="sxs-lookup"><span data-stu-id="49df8-135">In this case to handle international names on the local Intranet, the DNS servers that are used for the Intranet should support Unicode name resolution.</span></span>

- <span data-ttu-id="49df8-136">idn enabled = なし</span><span class="sxs-lookup"><span data-stu-id="49df8-136">idn enabled = None</span></span>

     <span data-ttu-id="49df8-137">この値では、Unicode ドメイン名を変換して Punycode を使用することはできません。</span><span class="sxs-lookup"><span data-stu-id="49df8-137">This value will not convert any Unicode domain names to use Punycode.</span></span> <span data-ttu-id="49df8-138">これは、.NET Framework 2.0 の動作と一貫性のある既定値です。</span><span class="sxs-lookup"><span data-stu-id="49df8-138">This is the default value which is consistent with the .NET Framework 2.0 behavior.</span></span>

 <span data-ttu-id="49df8-139">IDN を有効にすると、ドメイン名に含まれるすべての Unicode のラベルが Punycode のラベルに変換されます。</span><span class="sxs-lookup"><span data-stu-id="49df8-139">Enabling IDN will convert all Unicode labels in a domain name to their Punycode equivalents.</span></span> <span data-ttu-id="49df8-140">Punycode 名には ASCII 文字のみが含まれ、常に xn-- プレフィックスで始まります。</span><span class="sxs-lookup"><span data-stu-id="49df8-140">Punycode names contain only ASCII characters and always start with the xn-- prefix.</span></span> <span data-ttu-id="49df8-141">この理由は、ほとんどの DNS サーバーは ASCII 文字しかサポートしていないため、インターネットで既存の DNS サーバーをサポートするためです (RFC 3940 を参照)。</span><span class="sxs-lookup"><span data-stu-id="49df8-141">The reason for this is to support existing DNS servers on the Internet, since most DNS servers only support ASCII characters (see RFC 3940).</span></span>
