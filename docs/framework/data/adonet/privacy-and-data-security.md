## <a name="cryptography-and-hash-codes"></a><span data-ttu-id="6a2dc-105">暗号化とハッシュ コード</span><span class="sxs-lookup"><span data-stu-id="6a2dc-105">Cryptography and Hash Codes</span></span>  
 <span data-ttu-id="6a2dc-106">ADO.NET アプリケーションから .NET Framework <xref:System.Security.Cryptography> 名前空間のクラスを使用すると、認証されていない第三者によるデータの読み取りや変更を防止できます。</span><span class="sxs-lookup"><span data-stu-id="6a2dc-106">The classes in the .NET Framework <xref:System.Security.Cryptography> namespace can be used from your ADO.NET applications to prevent data from being read or modified by unauthorized third parties.</span></span> <span data-ttu-id="6a2dc-107">クラスには、アンマネージド Microsoft CryptoAPI 用のラッパーもあれば、マネージド実装もあります。</span><span class="sxs-lookup"><span data-stu-id="6a2dc-107">Some classes are wrappers for the unmanaged Microsoft CryptoAPI, while others are managed implementations.</span></span> <span data-ttu-id="6a2dc-108">「[暗号化サービス](../../../standard/security/cryptographic-services.md)」トピックでは、.NET Framework における暗号化の概要や暗号化の実装方法と、暗号化に関連した個々のタスクを実行する方法について説明されています。</span><span class="sxs-lookup"><span data-stu-id="6a2dc-108">The [Cryptographic Services](../../../standard/security/cryptographic-services.md) topic provides an overview of cryptography in the .NET Framework, describes how cryptograph is implemented, and how you can perform specific cryptographic tasks.</span></span>  
  
 <span data-ttu-id="6a2dc-109">暗号化ではデータを暗号化した後に復号化を行いますが、データのハッシュは不可逆性の処理です。</span><span class="sxs-lookup"><span data-stu-id="6a2dc-109">Unlike cryptography, which allows data to be encrypted and then decrypted, hashing data is a one-way process.</span></span> <span data-ttu-id="6a2dc-110">データのハッシュは、データが変更されていないことを確認することによって改ざんを防止する場合に役立ちます。ハッシュ アルゴリズムは、同一内容の入力文字列に対して、容易に比較できる同一内容の短い値を出力します。</span><span class="sxs-lookup"><span data-stu-id="6a2dc-110">Hashing data is useful when you want to prevent tampering by checking that data has not been altered: given identical input strings, hashing algorithms always produce identical short output values that can easily be compared.</span></span> <span data-ttu-id="6a2dc-111">「[ハッシュ コードによるデータの整合性の保証](../../../standard/security/ensuring-data-integrity-with-hash-codes.md)」では、ハッシュ値を生成および検証する方法について説明されています。</span><span class="sxs-lookup"><span data-stu-id="6a2dc-111">[Ensuring Data Integrity with Hash Codes](../../../standard/security/ensuring-data-integrity-with-hash-codes.md) describes how you can generate and verify hash values.</span></span>

## <a name="securing-string-values-in-memory"></a><span data-ttu-id="6a2dc-119">メモリ内の文字列値の保護</span><span class="sxs-lookup"><span data-stu-id="6a2dc-119">Securing String Values in Memory</span></span>  
 <span data-ttu-id="6a2dc-120"><xref:System.String> オブジェクトにパスワード、クレジット カード番号、個人データなどの機密情報が含まれていると、そのデータの使用後に、情報が漏洩するリスクが生じます。アプリケーションでは、文字列データをコンピューターのメモリから削除することはできません。</span><span class="sxs-lookup"><span data-stu-id="6a2dc-120">If a <xref:System.String> object contains sensitive information, such as a password, credit card number, or personal data, there is a risk that the information could be revealed after it is used because the application cannot delete the data from computer memory.</span></span>  
  
 <span data-ttu-id="6a2dc-121"><xref:System.String> は不変であり、いったん作成された文字列値は変更できません。</span><span class="sxs-lookup"><span data-stu-id="6a2dc-121">A <xref:System.String> is immutable; its value cannot be modified once it has been created.</span></span> <span data-ttu-id="6a2dc-122">文字列値を変更したように見えても、実際には <xref:System.String> オブジェクトの新しいインスタンスがメモリ内に作成されているだけです。データはテキスト形式で保存されます。</span><span class="sxs-lookup"><span data-stu-id="6a2dc-122">Changes that appear to modify the string value actually create a new instance of a <xref:System.String> object in memory, storing the data as plain text.</span></span> <span data-ttu-id="6a2dc-123">また、文字列のインスタンスがいつメモリから削除されるかを予測することも不可能です。</span><span class="sxs-lookup"><span data-stu-id="6a2dc-123">In addition, it is not possible to predict when the string instances will be deleted from memory.</span></span> <span data-ttu-id="6a2dc-124">文字列型によって占有されたメモリは、.NET のガベージ コレクションによって非確定的に回収されます。</span><span class="sxs-lookup"><span data-stu-id="6a2dc-124">Memory reclamation with strings is not deterministic with .NET garbage collection.</span></span> <span data-ttu-id="6a2dc-125">高い機密を要するデータに <xref:System.String> クラスや <xref:System.Text.StringBuilder> クラスを使用することは避けてください。</span><span class="sxs-lookup"><span data-stu-id="6a2dc-125">You should avoid using the <xref:System.String> and <xref:System.Text.StringBuilder> classes if your data is truly sensitive.</span></span>  
  
 <span data-ttu-id="6a2dc-126"><xref:System.Security.SecureString> クラスには、Data Protection API (DPAPI) を使ってメモリ内のテキストを暗号化するためのメソッドが用意されています。</span><span class="sxs-lookup"><span data-stu-id="6a2dc-126">The <xref:System.Security.SecureString> class provides methods for encrypting text using the Data Protection API (DPAPI) in memory.</span></span> <span data-ttu-id="6a2dc-127">文字列は不要になった段階でメモリから削除されます。</span><span class="sxs-lookup"><span data-stu-id="6a2dc-127">The string is then deleted from memory when it is no longer needed.</span></span> <span data-ttu-id="6a2dc-128">文字列の内容を簡単に読み取ることのできる `ToString` メソッドは、<xref:System.Security.SecureString> には存在しません。</span><span class="sxs-lookup"><span data-stu-id="6a2dc-128">There is no `ToString` method to quickly read the contents of a <xref:System.Security.SecureString>.</span></span> <span data-ttu-id="6a2dc-129">`SecureString` の新しいインスタンスを作成する際は、値を渡さずに初期化することも、<xref:System.Char> オブジェクトの配列へのポインターを渡して初期化することもできます。</span><span class="sxs-lookup"><span data-stu-id="6a2dc-129">You can initialize a new instance of `SecureString` with no value or by passing it a pointer to an array of <xref:System.Char> objects.</span></span> <span data-ttu-id="6a2dc-130">その後、このクラスの各種のメソッドを使って文字列を操作できます。</span><span class="sxs-lookup"><span data-stu-id="6a2dc-130">You can then use the various methods of the class to work with the string.</span></span>
