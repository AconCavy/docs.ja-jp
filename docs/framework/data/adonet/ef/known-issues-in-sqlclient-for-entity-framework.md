## <a name="skip-operator"></a><span data-ttu-id="f2947-121">SKIP 演算子</span><span class="sxs-lookup"><span data-stu-id="f2947-121">SKIP Operator</span></span>  
 <span data-ttu-id="f2947-122">SQL Server 2000 を使用している場合、キー以外の列で ORDER BY と共に SKIP を使用すると、不適切な結果が返される場合があります。</span><span class="sxs-lookup"><span data-stu-id="f2947-122">If you are using SQL Server 2000, using SKIP with ORDER BY on non-key columns might return incorrect results.</span></span> <span data-ttu-id="f2947-123">キー以外の列に重複するデータが存在する場合、指定された数を超える行はスキップされます。</span><span class="sxs-lookup"><span data-stu-id="f2947-123">More than the specified number of rows might be skipped if the non-key column has duplicate data in it.</span></span> <span data-ttu-id="f2947-124">これは、SQL Server 2000 での SKIP の変換方法によるものです。</span><span class="sxs-lookup"><span data-stu-id="f2947-124">This is due to how SKIP is translated for SQL Server 2000.</span></span> <span data-ttu-id="f2947-125">たとえば、次のクエリでは、`E.NonKeyColumn` に重複値が存在する場合、5 行を超える行はスキップされます。</span><span class="sxs-lookup"><span data-stu-id="f2947-125">For example, in the following query, more than five rows might be skipped if `E.NonKeyColumn` has duplicate values:</span></span>
