```sql
[ TOP (n) ]
```

## <a name="remarks"></a><span data-ttu-id="d91d4-109">Remarks</span><span class="sxs-lookup"><span data-stu-id="d91d4-109">Remarks</span></span>

<span data-ttu-id="d91d4-110">TOP 式には、単一の数値リテラルまたは単一のパラメーターを使用してください。</span><span class="sxs-lookup"><span data-stu-id="d91d4-110">The TOP expression must be either a single numeric literal or a single parameter.</span></span> <span data-ttu-id="d91d4-111">定数リテラルを使用する場合は、リテラルの種類を Edm.Int64 (byte、int16、int32、int64、または Edm.Int64 に昇格可能な型にマップされる任意のプロバイダー型) に暗黙的に昇格でき、その値が 0 以上である必要があります。</span><span class="sxs-lookup"><span data-stu-id="d91d4-111">If a constant literal is used, the literal type must be implicitly promotable to Edm.Int64 (byte, int16, int32 or int64 or any provider type that maps to a type that is promotable to Edm.Int64) and its value must be greater than or equal to zero.</span></span> <span data-ttu-id="d91d4-112">それ以外の場合は、例外が発生します。</span><span class="sxs-lookup"><span data-stu-id="d91d4-112">Otherwise an exception will be raised.</span></span> <span data-ttu-id="d91d4-113">パラメーターを式として使用する場合は、パラメーター型も Edm.Int64 に暗黙的に昇格できる必要がありますが、パラメーター値は遅延バインドされるため、コンパイル時に実際のパラメーター値は検証されません。</span><span class="sxs-lookup"><span data-stu-id="d91d4-113">If a parameter is used as an expression, the parameter type must also be implicitly promotable to Edm.Int64, but there will be no validation of the actual parameter value during compilation because the parameter values are late bounded.</span></span>

<span data-ttu-id="d91d4-114">定数 TOP 式の例を以下に示します。</span><span class="sxs-lookup"><span data-stu-id="d91d4-114">The following is an example of constant TOP expression:</span></span>


<span data-ttu-id="d91d4-115">パラメーター化された TOP 式の例を以下に示します。</span><span class="sxs-lookup"><span data-stu-id="d91d4-115">The following is an example of parameterized TOP expression:</span></span>

<span data-ttu-id="d91d4-116">TOP は、クエリが並べ替えられていない限り、非決定的です。</span><span class="sxs-lookup"><span data-stu-id="d91d4-116">TOP is non-deterministic unless the query is sorted.</span></span> <span data-ttu-id="d91d4-117">確定的な結果が必要な場合は、 [ORDER BY](skip-entity-sql.md) 句の [SKIP](limit-entity-sql.md) サブ句および [LIMIT](order-by-entity-sql.md) サブ句を使用します。</span><span class="sxs-lookup"><span data-stu-id="d91d4-117">If you require a deterministic result, use the [SKIP](skip-entity-sql.md) and [LIMIT](limit-entity-sql.md) sub-clauses in the [ORDER BY](order-by-entity-sql.md) clause.</span></span> <span data-ttu-id="d91d4-118">TOP     SKIP/LIMIT</span><span class="sxs-lookup"><span data-stu-id="d91d4-118">The TOP and SKIP/LIMIT are mutually exclusive.</span></span>
