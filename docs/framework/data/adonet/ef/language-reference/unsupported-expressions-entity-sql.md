## <a name="quantified-predicates"></a><span data-ttu-id="fe32c-105">定量化された述語</span><span class="sxs-lookup"><span data-stu-id="fe32c-105">Quantified predicates</span></span>

<span data-ttu-id="fe32c-106">Transact-SQL では、次の形式のコンストラクターを使用できます。</span><span class="sxs-lookup"><span data-stu-id="fe32c-106">Transact-SQL allows constructs of the following form:</span></span>

```sql
sal > all (select salary from employees)
sal > any (select salary from employees)
```

<span data-ttu-id="fe32c-107">ただし、[!INCLUDE[esql](../../../../../../includes/esql-md.md)] では、このようなコンストラクターがサポートされていません。</span><span class="sxs-lookup"><span data-stu-id="fe32c-107">[!INCLUDE[esql](../../../../../../includes/esql-md.md)], however, does not support such constructs.</span></span> <span data-ttu-id="fe32c-108">[!INCLUDE[esql](../../../../../../includes/esql-md.md)] では、これに相当する式を次のように記述できます。</span><span class="sxs-lookup"><span data-stu-id="fe32c-108">Equivalent expressions can be written in [!INCLUDE[esql](../../../../../../includes/esql-md.md)] as follows:</span></span>

```sql
not exists(select 0 from employees as e where sal <= e.salary)
exists(select 0 from employees as e where sal > e.salary)
```

## <a name="-operator"></a><span data-ttu-id="fe32c-109">\* 演算子</span><span class="sxs-lookup"><span data-stu-id="fe32c-109">\* operator</span></span>

<span data-ttu-id="fe32c-110">Transact-SQL では、すべての列を投影する必要があることを示すために、SELECT 句で \* 演算子を使用できます。これは [!INCLUDE[esql](../../../../../../includes/esql-md.md)] でサポートされていません。</span><span class="sxs-lookup"><span data-stu-id="fe32c-110">Transact-SQL supports the use of the \* operator in the SELECT clause to indicate that all columns should be projected out. This is not supported in [!INCLUDE[esql](../../../../../../includes/esql-md.md)].</span></span>
