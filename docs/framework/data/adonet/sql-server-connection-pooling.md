 <span data-ttu-id="3da0f-106">実際には、ほとんどのアプリケーションで、1 つまたはいくつかの違いがあるだけの構成を接続に使用しています。</span><span class="sxs-lookup"><span data-stu-id="3da0f-106">In practice, most applications use only one or a few different configurations for connections.</span></span> <span data-ttu-id="3da0f-107">つまり、アプリケーションの実行時に、多数の同一の接続が繰り返し開かれ、閉じられているのです。</span><span class="sxs-lookup"><span data-stu-id="3da0f-107">This means that during application execution, many identical connections will be repeatedly opened and closed.</span></span> <span data-ttu-id="3da0f-108">接続を開くコストを最小限にするため、ADO.NET では "*接続プール*" と呼ばれる最適化の手法が使われています。</span><span class="sxs-lookup"><span data-stu-id="3da0f-108">To minimize the cost of opening connections, ADO.NET uses an optimization technique called *connection pooling*.</span></span>

 <span data-ttu-id="3da0f-116">同じ構成を持つ接続のみをプールすることができます。</span><span class="sxs-lookup"><span data-stu-id="3da0f-116">Only connections with the same configuration can be pooled.</span></span> <span data-ttu-id="3da0f-117">ADO.NET では、同時に複数のプール (構成ごとに 1 つ) が保持されます。</span><span class="sxs-lookup"><span data-stu-id="3da0f-117">ADO.NET keeps several pools at the same time, one for each configuration.</span></span> <span data-ttu-id="3da0f-118">接続は接続文字列によってプール内で区別されます。また、統合セキュリティを使用している場合は、Windows ID によっても区別されます。</span><span class="sxs-lookup"><span data-stu-id="3da0f-118">Connections are separated into pools by connection string, and by Windows identity when integrated security is used.</span></span> <span data-ttu-id="3da0f-119">接続がプールされるかどうかは、その接続がトランザクションに参加しているかどうかにも依存します。</span><span class="sxs-lookup"><span data-stu-id="3da0f-119">Connections are also pooled based on whether they are enlisted in a transaction.</span></span> <span data-ttu-id="3da0f-120"><xref:System.Data.SqlClient.SqlConnection.ChangePassword%2A> を使用すると、<xref:System.Data.SqlClient.SqlCredential> インスタンスは接続プールに影響します。</span><span class="sxs-lookup"><span data-stu-id="3da0f-120">When using <xref:System.Data.SqlClient.SqlConnection.ChangePassword%2A>, the <xref:System.Data.SqlClient.SqlCredential> instance affects the connection pool.</span></span> <span data-ttu-id="3da0f-121">ユーザー ID とパスワードが同じでも、<xref:System.Data.SqlClient.SqlCredential> のインスタンスごとに異なる接続プールが使用されます。</span><span class="sxs-lookup"><span data-stu-id="3da0f-121">Different instances of <xref:System.Data.SqlClient.SqlCredential> will use different connection pools, even if the user ID and password are the same.</span></span>  
  
 <span data-ttu-id="3da0f-122">接続をプールすると、アプリケーションのパフォーマンスとスケーラビリティを大幅に改善できます。</span><span class="sxs-lookup"><span data-stu-id="3da0f-122">Pooling connections can significantly enhance the performance and scalability of your application.</span></span> <span data-ttu-id="3da0f-123">ADO.NET では、既定で接続プールが有効になります。</span><span class="sxs-lookup"><span data-stu-id="3da0f-123">By default, connection pooling is enabled in ADO.NET.</span></span> <span data-ttu-id="3da0f-124">明示的に無効にしない限り、プーラーは、アプリケーションで接続が開かれたり、閉じられたりする際に接続を最適化します。</span><span class="sxs-lookup"><span data-stu-id="3da0f-124">Unless you explicitly disable it, the pooler optimizes the connections as they are opened and closed in your application.</span></span> <span data-ttu-id="3da0f-125">また、接続プール機能の動作を制御する接続文字列修飾子を指定することもできます。</span><span class="sxs-lookup"><span data-stu-id="3da0f-125">You can also supply several connection string modifiers to control connection pooling behavior.</span></span> <span data-ttu-id="3da0f-126">詳細については、このトピックで後述する「接続文字列キーワードによる接続プールの制御」を参照してください。</span><span class="sxs-lookup"><span data-stu-id="3da0f-126">For more information, see "Controlling Connection Pooling with Connection String Keywords" later in this topic.</span></span>

## <a name="pool-creation-and-assignment"></a><span data-ttu-id="3da0f-130">プールの作成と割り当て</span><span class="sxs-lookup"><span data-stu-id="3da0f-130">Pool Creation and Assignment</span></span>  
 <span data-ttu-id="3da0f-131">接続が最初に開かれると、完全一致のアルゴリズムに基づいて接続プールが作成され、接続内の接続文字列に関連付けられます。</span><span class="sxs-lookup"><span data-stu-id="3da0f-131">When a connection is first opened, a connection pool is created based on an exact matching algorithm that associates the pool with the connection string in the connection.</span></span> <span data-ttu-id="3da0f-132">各接続プールが別の接続文字列に関連付けられます。</span><span class="sxs-lookup"><span data-stu-id="3da0f-132">Each connection pool is associated with a distinct connection string.</span></span> <span data-ttu-id="3da0f-133">新しい接続が開かれたとき、接続文字列が既存のプールと完全に一致しない場合は、新しいプールが作成されます。</span><span class="sxs-lookup"><span data-stu-id="3da0f-133">When a new connection is opened, if the connection string is not an exact match to an existing pool, a new pool is created.</span></span> <span data-ttu-id="3da0f-134">接続は、プロセス、アプリケーション ドメイン、接続文字列、Windows ID (統合セキュリティを使用している場合) ごとにプールされます。</span><span class="sxs-lookup"><span data-stu-id="3da0f-134">Connections are pooled per process, per application domain, per connection string and when integrated security is used, per Windows identity.</span></span> <span data-ttu-id="3da0f-135">また、接続文字列は完全に一致していることが必要です。同じ接続に対してキーワードを異なる順序で指定すると別々にプールされます。</span><span class="sxs-lookup"><span data-stu-id="3da0f-135">Connection strings must also be an exact match; keywords supplied in a different order for the same connection will be pooled separately.</span></span>  
  
 <span data-ttu-id="3da0f-136">3 つの新しい <xref:System.Data.SqlClient.SqlConnection> オブジェクトを作成し、2 つの接続プールだけでこの 3 つのオブジェクトを管理する C# の例を次に示します。</span><span class="sxs-lookup"><span data-stu-id="3da0f-136">In the following C# example, three new <xref:System.Data.SqlClient.SqlConnection> objects are created, but only two connection pools are required to manage them.</span></span> <span data-ttu-id="3da0f-137">1 番目の接続文字列と 2 番目の接続文字列では、`Initial Catalog` に割り当てる値が異なります。</span><span class="sxs-lookup"><span data-stu-id="3da0f-137">Note that the first and second connection strings differ by the value assigned for `Initial Catalog`.</span></span>

  "Integrated Security=SSPI;Initial Catalog=Northwind"))  
    {  
        connection.Open();
        // Pool A is created.  
    }

  "Integrated Security=SSPI;Initial Catalog=pubs"))  
    {  
        connection.Open();
        // Pool B is created because the connection strings differ.  
    }

  "Integrated Security=SSPI;Initial Catalog=Northwind"))

## <a name="adding-connections"></a><span data-ttu-id="3da0f-142">接続の追加</span><span class="sxs-lookup"><span data-stu-id="3da0f-142">Adding Connections</span></span>
