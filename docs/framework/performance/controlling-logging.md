### <a name="to-capture-clr-etw-events-using-logman"></a><span data-ttu-id="63393-127">Logman を使用して CLR ETW イベントをキャプチャするには</span><span class="sxs-lookup"><span data-stu-id="63393-127">To capture CLR ETW events using Logman</span></span>

     `logman start clrevents -p {e13c0d23-ccbc-4e12-931b-d9cc2eee27e4} 0x1CCBD 0x5 -ets -ct perf`

     <span data-ttu-id="63393-129">ここで、</span><span class="sxs-lookup"><span data-stu-id="63393-129">where:</span></span>

    - <span data-ttu-id="63393-130">`-p` パラメーターはプロバイダーの GUID を識別します。</span><span class="sxs-lookup"><span data-stu-id="63393-130">The `-p` parameter identifies the provider GUID.</span></span>

    - <span data-ttu-id="63393-131">`0x1CCBD` は、発生するイベントのカテゴリを指定しています。</span><span class="sxs-lookup"><span data-stu-id="63393-131">`0x1CCBD` specifies the categories of events that will be raised.</span></span>

    - <span data-ttu-id="63393-132">`0x5` は、ログ レベルを設定しています (この場合は詳細 (5))。</span><span class="sxs-lookup"><span data-stu-id="63393-132">`0x5` sets the level of logging (in this case, verbose (5)).</span></span>

    - <span data-ttu-id="63393-133">`-ets` パラメーターは、コマンドをイベント トレース セッションに送信するように指定します。</span><span class="sxs-lookup"><span data-stu-id="63393-133">The `-ets` parameter instructs Logman to send commands to event tracing sessions.</span></span>

    - <span data-ttu-id="63393-134">`-ct perf` パラメーターは、`QueryPerformanceCounter` 関数を使用して各イベントのタイム スタンプを記録するように指定します。</span><span class="sxs-lookup"><span data-stu-id="63393-134">The `-ct perf` parameter specifies that the `QueryPerformanceCounter` function will be used to log the time stamp for each event.</span></span>
