# <a name="type-argument-inference-failed-for-type-parameter-typeparametername-of-genericproceduresignature"></a><span data-ttu-id="b69af-102">'\<genericproceduresignature> の型パラメーター '\<typeparametername>' に対する型引数の推定が失敗しました</span><span class="sxs-lookup"><span data-stu-id="b69af-102">Type argument inference failed for type parameter '\<typeparametername>' of '\<genericproceduresignature>'</span></span>

<span data-ttu-id="b69af-103">**エラー ID:** BC32051</span><span class="sxs-lookup"><span data-stu-id="b69af-103">**Error ID:** BC32051</span></span>

<span data-ttu-id="b69af-104">'\<genericproceduresignature>' の型パラメーター '\<typeparametername>' に対する型引数の推定に失敗しました。</span><span class="sxs-lookup"><span data-stu-id="b69af-104">Type argument inference failed for type parameter '\<typeparametername>' of '\<genericproceduresignature>'.</span></span> <span data-ttu-id="b69af-105">型引数は、パラメーター '\<parametername>' に渡された引数から推測できませんでした。</span><span class="sxs-lookup"><span data-stu-id="b69af-105">Type argument could not be inferred from the argument passed to parameter '\<parametername>'.</span></span>

 <span data-ttu-id="b69af-106">ジェネリック プロシージャが型引数を指定せずに呼び出されたため、コンパイラはパラメーターのいずれかに渡す型を推定できません。</span><span class="sxs-lookup"><span data-stu-id="b69af-106">A generic procedure is called without supplying any type arguments, and the compiler cannot infer the type to pass to one of the parameters.</span></span>

 <span data-ttu-id="b69af-107">通常、ジェネリック プロシージャを呼び出す場合は、ジェネリック プロシージャで定義する型パラメーターごとに型引数を指定します。</span><span class="sxs-lookup"><span data-stu-id="b69af-107">Normally, when you call a generic procedure, you supply a type argument for each type parameter that the generic procedure defines.</span></span> <span data-ttu-id="b69af-108">型引数を指定しない場合、コンパイラは型パラメーターに渡す型を推定しようとします。</span><span class="sxs-lookup"><span data-stu-id="b69af-108">If you do not supply any type arguments, then the compiler attempts to infer the types to be passed to the type parameters.</span></span> <span data-ttu-id="b69af-109">呼び出しのコンテキストから型パラメーターについて競合するデータ型情報が提供される場合、型の推定は失敗します。</span><span class="sxs-lookup"><span data-stu-id="b69af-109">If the context of the call provides conflicting data type information for a type parameter, then the type inference fails.</span></span>

 <span data-ttu-id="b69af-110">次のコードでは、このエラーが発生する可能性があります。</span><span class="sxs-lookup"><span data-stu-id="b69af-110">The following code can generate this error:</span></span>

```vb
Public Sub DoSomething(Of T)(arg1 As T(), arg2 As T)
End Sub

Public Sub Main()
    DoSomething(6, 42)
End Sub
```  
  
 <span data-ttu-id="b69af-111">前の例では、コンパイラは `Integer` に渡される 42 という値に基づいて `t` に対する型 `arg2`を推定します。</span><span class="sxs-lookup"><span data-stu-id="b69af-111">In the preceding example, the compiler infers type `Integer` for `t` based on the value of 42 passed to `arg2`.</span></span> <span data-ttu-id="b69af-112">ただし、その推定には `arg1` が型 `Integer()`(つまり、 `Integer`の配列) である必要があり、 `arg1` に渡される値 6 はその型と一致しません。</span><span class="sxs-lookup"><span data-stu-id="b69af-112">However, that inference would require `arg1` to be of type `Integer()`, that is, an array of `Integer`, and the value 6 passed to `arg1` does not match that type.</span></span>

## <a name="to-correct-this-error"></a><span data-ttu-id="b69af-113">このエラーを解決するには</span><span class="sxs-lookup"><span data-stu-id="b69af-113">To correct this error</span></span>

- <span data-ttu-id="b69af-114">ジェネリック プロシージャに型引数を指定し、コンパイラが型引数を推定しなくて済むようにします。</span><span class="sxs-lookup"><span data-stu-id="b69af-114">Supply type arguments to the generic procedure, so that the compiler does not have to infer them.</span></span>

- <span data-ttu-id="b69af-115">型引数の型に一致する型で通常の引数を指定します。</span><span class="sxs-lookup"><span data-stu-id="b69af-115">Supply normal arguments with types that match those of the type arguments.</span></span>
