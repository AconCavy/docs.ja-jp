- <span data-ttu-id="de01d-154">**拡張ロジックのみを含めます。**</span><span class="sxs-lookup"><span data-stu-id="de01d-154">**Include only the extension logic.**</span></span> <span data-ttu-id="de01d-155">`My` 名前空間の拡張機能に含まれるロジックには、`My` 名前空間で必要な機能を公開するために必要なコードのみを含める必要があります。</span><span class="sxs-lookup"><span data-stu-id="de01d-155">The logic included in the `My` namespace extension should include only the code that is needed to expose the required functionality in the `My` namespace.</span></span> <span data-ttu-id="de01d-156">拡張機能は、ユーザー プロジェクト内にソース コードとして存在し、拡張機能コンポーネントを更新するとメンテナンス コストが高くなるため、可能な場合は回避する必要があります。</span><span class="sxs-lookup"><span data-stu-id="de01d-156">Because your extension will reside in user projects as source code, updating the extension component incurs a high maintenance cost and should be avoided if possible.</span></span>
- <span data-ttu-id="de01d-157">**プロジェクトの前提を最小限に抑えます。**</span><span class="sxs-lookup"><span data-stu-id="de01d-157">**Minimize project assumptions.**</span></span> <span data-ttu-id="de01d-158">`My` 名前空間の拡張機能を作成する場合は、一連の参照、プロジェクトレベルのインポート、または特定のコンパイラ設定 (たとえば、`Option Strict` オフ) を前提としないでください。</span><span class="sxs-lookup"><span data-stu-id="de01d-158">When you create your extensions of the `My` namespace, do not assume a set of references, project-level imports, or specific compiler settings (for example, `Option Strict` off).</span></span> <span data-ttu-id="de01d-159">代わりに、`Global` キーワードを使用して、依存関係を最小限に抑え、すべての型参照を完全修飾してください。</span><span class="sxs-lookup"><span data-stu-id="de01d-159">Instead, minimize dependencies and fully qualify all type references by using the `Global` keyword.</span></span> <span data-ttu-id="de01d-160">また、拡張機能のエラーを最小限に抑えるために、拡張機能が `Option Strict` オンでコンパイルされるようにしてください。</span><span class="sxs-lookup"><span data-stu-id="de01d-160">Also, ensure that the extension compiles with `Option Strict` on to minimize errors in the extension.</span></span>
- <span data-ttu-id="de01d-161">**拡張機能のコードを分離します。**</span><span class="sxs-lookup"><span data-stu-id="de01d-161">**Isolate the extension code.**</span></span> <span data-ttu-id="de01d-162">コードを 1 つのファイルに配置すると、拡張機能を Visual Studio 項目テンプレートとして簡単に配置できるようになります。</span><span class="sxs-lookup"><span data-stu-id="de01d-162">Placing the code in a single file makes your extension easily deployable as a Visual Studio item template.</span></span> <span data-ttu-id="de01d-163">詳細については、このトピックの後半の「拡張機能のパッケージ化と配置」を参照してください。</span><span class="sxs-lookup"><span data-stu-id="de01d-163">For more information, see "Packaging and Deploying Extensions" later in this topic.</span></span> <span data-ttu-id="de01d-164">すべての `My` 名前空間拡張コードを 1 つのファイルまたはプロジェクト内の別のフォルダーに配置すると、ユーザーが `My` 名前空間の拡張機能を見つけることができます。</span><span class="sxs-lookup"><span data-stu-id="de01d-164">Placing all the `My` namespace extension code in a single file or a separate folder in a project will also help users locate the `My` namespace extension.</span></span>

## <a name="designing-class-libraries-for-my"></a><span data-ttu-id="de01d-165">`My` のクラス ライブラリの設計</span><span class="sxs-lookup"><span data-stu-id="de01d-165">Designing class libraries for `My`</span></span>

<span data-ttu-id="de01d-166">ほとんどのオブジェクト モデルの場合と同様に、設計パターンの中には `My` 名前空間で適切に機能するものもあれば、そうでないものもあります。</span><span class="sxs-lookup"><span data-stu-id="de01d-166">As is the case with most object models, some design patterns work well in the `My` namespace and others do not.</span></span> <span data-ttu-id="de01d-167">`My` 名前空間の拡張機能を設計するときは、次の原則を考慮してください。</span><span class="sxs-lookup"><span data-stu-id="de01d-167">When designing an extension to the `My` namespace, consider the following principles:</span></span>

- <span data-ttu-id="de01d-168">**ステートレス メソッド。**</span><span class="sxs-lookup"><span data-stu-id="de01d-168">**Stateless methods.**</span></span> <span data-ttu-id="de01d-169">`My` 名前空間のメソッドは、特定のタスクに完全なソリューションを提供する必要があります。</span><span class="sxs-lookup"><span data-stu-id="de01d-169">Methods in the `My` namespace should provide a complete solution to a specific task.</span></span> <span data-ttu-id="de01d-170">メソッドに渡されたパラメーター値が、特定のタスクを完了するために必要なすべての入力を提供していることを確認してください。</span><span class="sxs-lookup"><span data-stu-id="de01d-170">Ensure that the parameter values that are passed to the method provide all the input required to complete the particular task.</span></span> <span data-ttu-id="de01d-171">リソースへのオープン接続など、以前の状態に依存するメソッドを作成しないようにしてください。</span><span class="sxs-lookup"><span data-stu-id="de01d-171">Avoid creating methods that rely on prior state, such as open connections to resources.</span></span>
- <span data-ttu-id="de01d-172">**グローバル インスタンス。**</span><span class="sxs-lookup"><span data-stu-id="de01d-172">**Global instances.**</span></span> <span data-ttu-id="de01d-173">`My` 名前空間で保持される唯一の状態は、プロジェクトに対してグローバルです。</span><span class="sxs-lookup"><span data-stu-id="de01d-173">The only state that is maintained in the `My` namespace is global to the project.</span></span> <span data-ttu-id="de01d-174">たとえば、`My.Application.Info` は、アプリケーション全体で共有される状態をカプセル化します。</span><span class="sxs-lookup"><span data-stu-id="de01d-174">For example, `My.Application.Info` encapsulates state that is shared throughout the application.</span></span>
- <span data-ttu-id="de01d-175">**単純なパラメーターの型。**</span><span class="sxs-lookup"><span data-stu-id="de01d-175">**Simple parameter types.**</span></span> <span data-ttu-id="de01d-176">複雑なパラメーターの型を避け、シンプルに保ってください。</span><span class="sxs-lookup"><span data-stu-id="de01d-176">Keep things simple by avoiding complex parameter types.</span></span> <span data-ttu-id="de01d-177">代わりに、パラメーター入力を受け取らないメソッド、または文字列、プリミティブ型などの単純な入力型を受け取るメソッドを作成してください。</span><span class="sxs-lookup"><span data-stu-id="de01d-177">Instead, create methods that either take no parameter input or that take simple input types such as strings, primitive types, and so on.</span></span>
- <span data-ttu-id="de01d-178">**ファクトリ メソッド。**</span><span class="sxs-lookup"><span data-stu-id="de01d-178">**Factory methods.**</span></span> <span data-ttu-id="de01d-179">一部の型は、インスタンス化が困難な場合があります。</span><span class="sxs-lookup"><span data-stu-id="de01d-179">Some types are necessarily difficult to instantiate.</span></span> <span data-ttu-id="de01d-180">ファクトリ メソッドを `My` 名前空間の拡張機能として指定すると、このカテゴリに分類される型をより簡単に検出して使用できます。</span><span class="sxs-lookup"><span data-stu-id="de01d-180">Providing factory methods as extensions to the `My` namespace enables you to more easily discover and consume types that fall into this category.</span></span> <span data-ttu-id="de01d-181">適切に機能するファクトリ メソッドの例として、`My.Computer.FileSystem.OpenTextFileReader` があります。</span><span class="sxs-lookup"><span data-stu-id="de01d-181">An example of a factory method that works well is `My.Computer.FileSystem.OpenTextFileReader`.</span></span> <span data-ttu-id="de01d-182">.NET Framework で使用できるいくつかのストリーム型があります。</span><span class="sxs-lookup"><span data-stu-id="de01d-182">There are several stream types available in the .NET Framework.</span></span> <span data-ttu-id="de01d-183">具体的にテキスト ファイルを指定することによって、`OpenTextFileReader` によりユーザーが使用するストリームを理解できます。</span><span class="sxs-lookup"><span data-stu-id="de01d-183">By specifying text files specifically, the `OpenTextFileReader` helps the user understand which stream to use.</span></span>

<span data-ttu-id="de01d-184">これらのガイドラインでは、クラス ライブラリの一般的な設計原則は使用できません。</span><span class="sxs-lookup"><span data-stu-id="de01d-184">These guidelines do not preclude general design principles for class libraries.</span></span> <span data-ttu-id="de01d-185">むしろ、これらは、Visual Basic と `My` 名前空間を使用している開発者向けに最適化された推奨事項です。</span><span class="sxs-lookup"><span data-stu-id="de01d-185">Rather, they are recommendations that are optimized for developers who are using Visual Basic and the `My` namespace.</span></span> <span data-ttu-id="de01d-186">クラス ライブラリを作成するための一般的な設計原則については、「[フレームワーク デザインのガイドライン](../../../standard/design-guidelines/index.md)」を参照してください。</span><span class="sxs-lookup"><span data-stu-id="de01d-186">For general design principles for creating class libraries, see [Framework Design Guidelines](../../../standard/design-guidelines/index.md).</span></span>
