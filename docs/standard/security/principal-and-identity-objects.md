## <a name="identity-objects"></a><span data-ttu-id="e1257-109">ID オブジェクト</span><span class="sxs-lookup"><span data-stu-id="e1257-109">Identity Objects</span></span>

 <span data-ttu-id="e1257-115">インターフェイスは、 <xref:System.Security.Principal.IIdentity> 名前と認証の種類 (Kerberos V5 や NTLM など) にアクセスするためのプロパティを定義します。</span><span class="sxs-lookup"><span data-stu-id="e1257-115">The <xref:System.Security.Principal.IIdentity> interface defines properties for accessing a name and an authentication type, such as Kerberos V5 or NTLM.</span></span> <span data-ttu-id="e1257-116">すべての **Identity** クラスでは **IIdentity** インターフェイスが実装されます。</span><span class="sxs-lookup"><span data-stu-id="e1257-116">All **Identity** classes implement the **IIdentity** interface.</span></span> <span data-ttu-id="e1257-117">**ID** オブジェクトと、スレッドが現在実行している Windows NT プロセス トークンの間に関係は必要ありません。</span><span class="sxs-lookup"><span data-stu-id="e1257-117">There is no required relationship between an **Identity** object and the Windows NT process token under which a thread is currently executing.</span></span> <span data-ttu-id="e1257-118">ただし、**ID** オブジェクトが **WindowsIdentity** オブジェクトである場合、ID は Windows NT セキュリティ トークンを表すと見なされます。</span><span class="sxs-lookup"><span data-stu-id="e1257-118">However, if the **Identity** object is a **WindowsIdentity** object, the identity is assumed to represent a Windows NT security token.</span></span>
