## <a name="guidelines"></a><span data-ttu-id="d4ac4-123">ガイドライン</span><span class="sxs-lookup"><span data-stu-id="d4ac4-123">Guidelines</span></span>

### <a name="choosing-the-right-serialization-technology-to-support"></a><span data-ttu-id="d4ac4-126">サポートする適切なシリアル化テクノロジの選択</span><span class="sxs-lookup"><span data-stu-id="d4ac4-126">Choosing the right serialization technology to support</span></span>

- <span data-ttu-id="d4ac4-129">型をシリアル化したときに作成される XML 形式の制御を強化する必要がある場合は、データ コントラクトのシリアル化の代わり、またはこれに加えて *XML シリアル化*をサポートすることを検討してください。</span><span class="sxs-lookup"><span data-stu-id="d4ac4-129">CONSIDER supporting the *XML serialization* instead of or in addition to data contract serialization if you need more control over the XML format that is produced when the type is serialized.</span></span>  
  
     <span data-ttu-id="d4ac4-130">これは、データ コントラクトのシリアル化でサポートされていない XML 構築を使用して XML 属性を作成するような一部の相互運用シナリオで必要になることがあります。</span><span class="sxs-lookup"><span data-stu-id="d4ac4-130">This may be necessary in some interoperability scenarios where you need to use an XML construct that is not supported by data contract serialization, for example, to produce XML attributes.</span></span>  
  
- <span data-ttu-id="d4ac4-131">使用する型のインスタンスが .NET リモート処理境界を超えて移動する必要がある場合は、*ランタイムのシリアル化*をサポートすることを検討してください。</span><span class="sxs-lookup"><span data-stu-id="d4ac4-131">CONSIDER supporting *runtime serialization* if instances of your type need to travel across .NET Remoting boundaries.</span></span>  
  
- <span data-ttu-id="d4ac4-132">一般的な永続化のためにランタイムのシリアル化や XML のシリアル化をサポートしないでください。</span><span class="sxs-lookup"><span data-stu-id="d4ac4-132">AVOID supporting runtime serialization or XML serialization just for general persistence reasons.</span></span> <span data-ttu-id="d4ac4-133">データ コントラクトのシリアル化を優先的に使用してください。</span><span class="sxs-lookup"><span data-stu-id="d4ac4-133">Prefer data contract serialization instead</span></span>
