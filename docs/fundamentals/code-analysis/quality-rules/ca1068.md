## <a name="rule-description"></a><span data-ttu-id="f9ce4-113">規則の説明</span><span class="sxs-lookup"><span data-stu-id="f9ce4-113">Rule description</span></span>

<span data-ttu-id="f9ce4-114">実行時間の長い操作や非同期操作を実行し、取り消し可能なメソッドは通常、キャンセルトークンパラメーターを受け取ります。</span><span class="sxs-lookup"><span data-stu-id="f9ce4-114">Methods that perform long running operations or asynchronous operations and are cancelable normally take a cancellation token parameter.</span></span> <span data-ttu-id="f9ce4-115">各キャンセルトークンには、 <xref:System.Threading.CancellationTokenSource> トークンを作成し、キャンセル可能な計算に使用するがあります。</span><span class="sxs-lookup"><span data-stu-id="f9ce4-115">Each cancellation token has a <xref:System.Threading.CancellationTokenSource> that creates the token and uses it for cancelable computations.</span></span> <span data-ttu-id="f9ce4-116">一般的な方法として、キャンセルトークンを呼び出し元から呼び出し先に渡すメソッド呼び出しの長いチェーンを用意します。</span><span class="sxs-lookup"><span data-stu-id="f9ce4-116">It is common practice to have a long chain of method calls that pass around the cancellation token from the callers to the callees.</span></span> <span data-ttu-id="f9ce4-117">したがって、キャンセル可能な計算に含まれる多数のメソッドは、キャンセルトークンパラメーターを持つことになります。</span><span class="sxs-lookup"><span data-stu-id="f9ce4-117">Hence, a large number of methods that take part in a cancelable computation end up having a cancellation token parameter.</span></span> <span data-ttu-id="f9ce4-118">ただし、キャンセルトークン自体は、通常、これらのメソッドの大部分のコア機能に関連しているわけではありません。</span><span class="sxs-lookup"><span data-stu-id="f9ce4-118">However, the cancellation token itself is not usually relevant to the core functionality of a majority of these methods.</span></span> <span data-ttu-id="f9ce4-119">このようなパラメーターをリストの最後のパラメーターとして使用することをお勧めします。</span><span class="sxs-lookup"><span data-stu-id="f9ce4-119">It's considered a good API design practice to have such parameters be the last parameter in the list.</span></span>
