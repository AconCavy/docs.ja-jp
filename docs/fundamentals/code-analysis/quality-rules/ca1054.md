## <a name="cause"></a><span data-ttu-id="b169f-111">原因</span><span class="sxs-lookup"><span data-stu-id="b169f-111">Cause</span></span>

<span data-ttu-id="b169f-112">型は、名前に "uri"、"Uri"、"urn"、"Urn"、"url"、または "Url" を含む文字列パラメーターを持つメソッドを宣言します。型は、パラメーターを受け取る対応するオーバーロードを宣言しません <xref:System.Uri?displayProperty=fullName> 。</span><span class="sxs-lookup"><span data-stu-id="b169f-112">A type declares a method with a string parameter whose name contains "uri", "Uri", "urn", "Urn", "url", or "Url" and the type does not declare a corresponding overload that takes a <xref:System.Uri?displayProperty=fullName> parameter.</span></span>

<span data-ttu-id="b169f-113">既定では、この規則は外部から参照できる型のみを参照しますが、これは [構成可能](#configure-code-to-analyze)です。</span><span class="sxs-lookup"><span data-stu-id="b169f-113">By default, this rule only looks at externally visible types, but this is [configurable](#configure-code-to-analyze).</span></span>

## <a name="rule-description"></a><span data-ttu-id="b169f-114">規則の説明</span><span class="sxs-lookup"><span data-stu-id="b169f-114">Rule description</span></span>

<span data-ttu-id="b169f-115">このルールでは、camel 形式の表記規則に基づいてパラメーター名をトークンに分割し、各トークンが "uri"、"Uri"、"urn"、"Urn"、"url"、または "Url" に等しいかどうかを確認します。</span><span class="sxs-lookup"><span data-stu-id="b169f-115">This rule splits the parameter name into tokens based on the camel casing convention and checks whether each token equals "uri", "Uri", "urn", "Urn", "url", or "Url".</span></span> <span data-ttu-id="b169f-116">一致するものがある場合、このルールは、パラメーターが URI (uniform resource identifier) を表すことを前提としています。</span><span class="sxs-lookup"><span data-stu-id="b169f-116">If there is a match, the rule assumes that the parameter represents a uniform resource identifier (URI).</span></span> <span data-ttu-id="b169f-117">URI の文字列表現は解析エラーやエンコーディング エラーが発生しやすく、セキュリティ上の脆弱性の原因となる場合があります。</span><span class="sxs-lookup"><span data-stu-id="b169f-117">A string representation of a URI is prone to parsing and encoding errors, and can lead to security vulnerabilities.</span></span> <span data-ttu-id="b169f-118">メソッドが URI の文字列表現を受け取る場合は、対応するオーバーロードを指定して、クラスのインスタンスを取得する必要があり <xref:System.Uri> ます。これにより、これらのサービスが安全かつ安全な方法で提供されます。</span><span class="sxs-lookup"><span data-stu-id="b169f-118">If a method takes a string representation of a URI, a corresponding overload should be provided that takes an instance of the <xref:System.Uri> class, which provides these services in a safe and secure manner.</span></span>

## <a name="how-to-fix-violations"></a><span data-ttu-id="b169f-119">違反の修正方法</span><span class="sxs-lookup"><span data-stu-id="b169f-119">How to fix violations</span></span>

<span data-ttu-id="b169f-120">この規則違反を修正するには、パラメーターを型に変更 <xref:System.Uri> します。これは互換性に影響する変更です。</span><span class="sxs-lookup"><span data-stu-id="b169f-120">To fix a violation of this rule, change the parameter to a <xref:System.Uri> type; this is a breaking change.</span></span> <span data-ttu-id="b169f-121">または、パラメーターを受け取るメソッドのオーバーロードを指定し <xref:System.Uri> ます。これは、非互換性に影響する変更です。</span><span class="sxs-lookup"><span data-stu-id="b169f-121">Alternately, provide an overload of the method that takes a <xref:System.Uri> parameter; this is a non-breaking change.</span></span>
