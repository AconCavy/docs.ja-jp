## <a name="create-the-data-model"></a><span data-ttu-id="effa1-108">データ モデルの作成</span><span class="sxs-lookup"><span data-stu-id="effa1-108">Create the data model</span></span>

<span data-ttu-id="effa1-116">列は次のように読み込みます。</span><span class="sxs-lookup"><span data-stu-id="effa1-116">Load columns as:</span></span>

<span data-ttu-id="effa1-119">ベクター プロパティがある場合は、データ モデルのプロパティに [`VectorType`](xref:Microsoft.ML.Data.VectorTypeAttribute) 属性を適用します。</span><span class="sxs-lookup"><span data-stu-id="effa1-119">If you have a vector property, apply the [`VectorType`](xref:Microsoft.ML.Data.VectorTypeAttribute) attribute to the property in your data model.</span></span> <span data-ttu-id="effa1-120">ベクター内のすべての要素は同じ型にする必要がある点に注意してください。</span><span class="sxs-lookup"><span data-stu-id="effa1-120">It's important to note that all of the elements in the vector need to be the same type.</span></span> <span data-ttu-id="effa1-121">列を分割したままにすると、特徴エンジニアリングが容易になり、柔軟性が向上しますが、列数が非常に多い場合、個々の列を操作するとトレーニング速度に影響します。</span><span class="sxs-lookup"><span data-stu-id="effa1-121">Keeping the columns separated allows for ease and flexibility of feature engineering, but for a very large number of columns, operating on the individual columns causes an impact on training speed.</span></span>

<span data-ttu-id="effa1-122">ML.NET は列名を介して動作します。</span><span class="sxs-lookup"><span data-stu-id="effa1-122">ML.NET Operates through column names.</span></span> <span data-ttu-id="effa1-123">列の名前をプロパティ名以外に変更する場合は、[`ColumnName`](xref:Microsoft.ML.Data.ColumnNameAttribute) 属性を使用します。</span><span class="sxs-lookup"><span data-stu-id="effa1-123">If you want to change the name of a column to something other than the property name, use the [`ColumnName`](xref:Microsoft.ML.Data.ColumnNameAttribute) attribute.</span></span> <span data-ttu-id="effa1-124">メモリ内オブジェクトを作成するときも、プロパティ名を使ってオブジェクトを作成します。</span><span class="sxs-lookup"><span data-stu-id="effa1-124">When creating in-memory objects, you still create objects using the property name.</span></span> <span data-ttu-id="effa1-125">ただし、データ処理と機械学習モデルの構築の場合、ML.NET では [`ColumnName`](xref:Microsoft.ML.Data.ColumnNameAttribute) 属性に指定された値でプロパティがオーバーライドされ、参照されます。</span><span class="sxs-lookup"><span data-stu-id="effa1-125">However, for data processing and building machine learning models, ML.NET overrides and references the property with the value provided in the [`ColumnName`](xref:Microsoft.ML.Data.ColumnNameAttribute) attribute.</span></span>

## <a name="load-data-from-a-single-file"></a><span data-ttu-id="effa1-126">1 つのファイルからデータを読み込む</span><span class="sxs-lookup"><span data-stu-id="effa1-126">Load data from a single file</span></span>

<span data-ttu-id="effa1-127">ファイルからデータを読み込むには、読み込むデータのデータ モデルと共に [`LoadFromTextFile`](xref:Microsoft.ML.TextLoaderSaverCatalog.LoadFromTextFile*) メソッドを使用します。</span><span class="sxs-lookup"><span data-stu-id="effa1-127">To load data from a file use the [`LoadFromTextFile`](xref:Microsoft.ML.TextLoaderSaverCatalog.LoadFromTextFile*) method along with the data model for the data to be loaded.</span></span> <span data-ttu-id="effa1-128">`separatorChar` パラメーターは既定でタブ区切りなので、必要に応じてデータ ファイルに合わせて変更します。</span><span class="sxs-lookup"><span data-stu-id="effa1-128">Since `separatorChar` parameter is tab-delimited by default, change it for your data file as needed.</span></span> <span data-ttu-id="effa1-129">ファイルにヘッダーがある場合は、ファイルの最初の行を無視して 2 行目からデータの読み込みを開始するように、`hasHeader` パラメーターを `true` に設定します。</span><span class="sxs-lookup"><span data-stu-id="effa1-129">If your file has a header, set the `hasHeader` parameter to `true` to ignore the first line in the file and begin to load data from the second line.</span></span>

## <a name="load-data-from-a-relational-database"></a><span data-ttu-id="effa1-137">リレーショナル データベースからデータを読み込む</span><span class="sxs-lookup"><span data-stu-id="effa1-137">Load data from a relational database</span></span>

<span data-ttu-id="effa1-138">ML.NET では、SQL Server、Azure SQL Database、Oracle、SQLite、PostgreSQL、Progress、IBM DB2 など数多くの、[`System.Data`](xref:System.Data) でサポートされている多様なリレーショナル データベースからのデータの読み込みがサポートされています。</span><span class="sxs-lookup"><span data-stu-id="effa1-138">ML.NET supports loading data from a variety of relational databases supported by [`System.Data`](xref:System.Data) that include SQL Server, Azure SQL Database, Oracle, SQLite, PostgreSQL, Progress, IBM DB2, and many more.</span></span>

> [!NOTE]

<span data-ttu-id="effa1-143">接続文字列と、データベースで実行される SQL コマンドを定義して、`DatabaseSource` インスタンスを作成します。</span><span class="sxs-lookup"><span data-stu-id="effa1-143">Define your connection string as well as the SQL command to be executed on the database and create a `DatabaseSource` instance.</span></span> <span data-ttu-id="effa1-144">このサンプルでは、LocalDB の SQL Server データベースをファイル パスと共に使用します。</span><span class="sxs-lookup"><span data-stu-id="effa1-144">This sample uses a LocalDB SQL Server database with a file path.</span></span> <span data-ttu-id="effa1-145">ただし、DatabaseLoader では、オンプレミスとクラウドのデータベースに対して、その他のすべての有効な接続文字列がサポートされています。</span><span class="sxs-lookup"><span data-stu-id="effa1-145">However, DatabaseLoader supports any other valid connection string for databases on-premises and in the cloud.</span></span>
