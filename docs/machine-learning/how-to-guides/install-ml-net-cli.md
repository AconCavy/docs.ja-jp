# <a name="how-to-install-the-mlnet-command-line-interface-cli-tool"></a><span data-ttu-id="e41c7-103">ML.NET コマンドライン インターフェイス (CLI) ツールをインストールする方法</span><span class="sxs-lookup"><span data-stu-id="e41c7-103">How to install the ML.NET Command-Line Interface (CLI) tool</span></span>

<span data-ttu-id="e41c7-104">ML.NET CLI (コマンドライン インターフェイス) は、用意したトレーニング データセットに基づいて高品質の ML.NET モデルとソース コードを生成するための、コマンドプロンプト (Windows、Mac、または Linux) 上で実行できるツールです。</span><span class="sxs-lookup"><span data-stu-id="e41c7-104">The ML.NET CLI (command-line interface) is a tool you can run on any command-prompt (Windows, Mac, or Linux) for generating good quality ML.NET models and source code based on training datasets you provide.</span></span>

> [!NOTE]
> <span data-ttu-id="e41c7-105">このトピックは、現在プレビュー段階の ML.NET CLI と ML.NET AutoML について述べており、内容が変更される場合があります。</span><span class="sxs-lookup"><span data-stu-id="e41c7-105">This topic refers to ML.NET CLI and ML.NET AutoML, which are currently in Preview, and material may be subject to change.</span></span>

<span data-ttu-id="e41c7-109">生成された C# コード プロジェクトは、Visual Studio で F5 キーまたは `dotnet run` (.NET Core CLI) を使用して実行できます。</span><span class="sxs-lookup"><span data-stu-id="e41c7-109">You can either run the generated C# code projects with Visual Studio F5 or with `dotnet run` (.NET Core CLI).</span></span>

## <a name="install"></a><span data-ttu-id="e41c7-111">インストール</span><span class="sxs-lookup"><span data-stu-id="e41c7-111">Install</span></span>

<span data-ttu-id="e41c7-112">ML.NET CLI のインストール方法は他の .NET グローバル ツールと同様です。</span><span class="sxs-lookup"><span data-stu-id="e41c7-112">The ML.NET CLI is installed like any other dotnet Global Tool.</span></span> <span data-ttu-id="e41c7-113">`dotnet tool install` .NET Core CLI コマンドを使用します。</span><span class="sxs-lookup"><span data-stu-id="e41c7-113">You use the `dotnet tool install` .NET Core CLI command.</span></span>

<span data-ttu-id="e41c7-115">ツールをインストールできない場合 (つまり、既定の NuGet フィードで利用できない場合) は、エラー メッセージが表示されます。</span><span class="sxs-lookup"><span data-stu-id="e41c7-115">If the tool can't be installed (that is, if it is not available at the default NuGet feed), error messages are displayed.</span></span> <span data-ttu-id="e41c7-116">予定していたフィードがチェックされていることを確認します。</span><span class="sxs-lookup"><span data-stu-id="e41c7-116">Check that the feeds you expected are being checked.</span></span>

<span data-ttu-id="e41c7-117">インストールが成功すると、ツールの呼び出しに使用したコマンドとインストールされたバージョンを示す、次の例のようなメッセージが表示されます。</span><span class="sxs-lookup"><span data-stu-id="e41c7-117">If installation is successful, a message is displayed showing the command used to call the tool and the version installed, similar to the following example:</span></span>

<span data-ttu-id="e41c7-118">インストールが成功したことを確認するには、次のコマンドを入力します。</span><span class="sxs-lookup"><span data-stu-id="e41c7-118">You can confirm the installation was successful by typing the following command:</span></span>

<span data-ttu-id="e41c7-119">"auto-train" コマンドなど、mlnet ツールに利用できるコマンドのヘルプを参照することをお勧めします。</span><span class="sxs-lookup"><span data-stu-id="e41c7-119">You should see the help for available commands for the mlnet tool such as the 'auto-train' command.</span></span>

## <a name="set-up-cli-suggestions-tab-based-auto-completion"></a><span data-ttu-id="e41c7-127">CLI の候補を設定する (タブベースのオートコンプリート)</span><span class="sxs-lookup"><span data-stu-id="e41c7-127">Set up CLI suggestions (tab-based auto-completion)</span></span>

<span data-ttu-id="e41c7-128">ML.NET CLI は `System.CommandLine` に基づいているので、タブ補完のサポートが組み込まれています。</span><span class="sxs-lookup"><span data-stu-id="e41c7-128">Since the ML.NET CLI is based on `System.CommandLine`, it has built-in support for tab completion.</span></span>

<span data-ttu-id="e41c7-129">次のアニメーションでタブのオートコンプリートのしくみの例を示します。</span><span class="sxs-lookup"><span data-stu-id="e41c7-129">An example of how tab auto completion works is shown in the following animation:</span></span>

![イメージ](./media/cli-tab-completion.gif)

<span data-ttu-id="e41c7-131">"タブベースのオートコンプリート" (パラメーターの候補) は *Windows PowerShell* と *macOS/Linux bash* では機能しますが、*Windows CMD* では機能しません。</span><span class="sxs-lookup"><span data-stu-id="e41c7-131">'Tab-based auto-completion' (parameter suggestions) works on *Windows PowerShell* and *macOS/Linux bash* but it won't work on *Windows CMD*.</span></span>

<span data-ttu-id="e41c7-132">これを有効にするには、現在のプレビュー バージョンでは、エンド ユーザーが、シェルごとに 1 回、以下に示すいくつかの手順を実行する必要があります。</span><span class="sxs-lookup"><span data-stu-id="e41c7-132">To enable it, in the current preview version, the end user has to take a few steps once per shell, outlined below.</span></span> <span data-ttu-id="e41c7-133">これが完了すると、ML.NET CLI などの `System.CommandLine` を使用して作成されるすべてのアプリに対して補完が機能するようになります。</span><span class="sxs-lookup"><span data-stu-id="e41c7-133">Once this is done, completions will work for all apps written using `System.CommandLine` such as the ML.NET CLI.</span></span>

<span data-ttu-id="e41c7-134">補完を有効にするマシンでは、2 つのことを実行する必要があります。</span><span class="sxs-lookup"><span data-stu-id="e41c7-134">On the machine where you'd like to enable completion, you'll need to do two things.</span></span>

    - <span data-ttu-id="e41c7-139">bash の場合、[dotnet-suggest-shim.bash](https://github.com/dotnet/System.CommandLine/blob/master/src/System.CommandLine.Suggest/dotnet-suggest-shim.bash) の内容を `~/.bash_profile` に追加します。</span><span class="sxs-lookup"><span data-stu-id="e41c7-139">For bash, add the contents of [dotnet-suggest-shim.bash](https://github.com/dotnet/System.CommandLine/blob/master/src/System.CommandLine.Suggest/dotnet-suggest-shim.bash) to `~/.bash_profile`.</span></span>

    - <span data-ttu-id="e41c7-140">PowerShell の場合は、[dotnet-recommend-shim.ps1](https://github.com/dotnet/System.CommandLine/blob/master/src/System.CommandLine.Suggest/dotnet-suggest-shim.ps1) の内容を PowerShell プロファイルに追加します。</span><span class="sxs-lookup"><span data-stu-id="e41c7-140">For PowerShell, add the contents of [dotnet-suggest-shim.ps1](https://github.com/dotnet/System.CommandLine/blob/master/src/System.CommandLine.Suggest/dotnet-suggest-shim.ps1) to your PowerShell profile.</span></span> <span data-ttu-id="e41c7-141">コンソールで次のコマンドを実行して、PowerShell プロファイルへの予想されるパスを見つけることができます。</span><span class="sxs-lookup"><span data-stu-id="e41c7-141">You can find the expected path to your PowerShell profile by running the following command in your console:</span></span>


## <a name="installation-directory"></a><span data-ttu-id="e41c7-143">インストール ディレクトリ</span><span class="sxs-lookup"><span data-stu-id="e41c7-143">Installation directory</span></span>

<span data-ttu-id="e41c7-144">ML.NET CLI は、既定のディレクトリまたは特定の場所にインストールできます。</span><span class="sxs-lookup"><span data-stu-id="e41c7-144">The ML.NET CLI can be installed in the default directory or in a specific location.</span></span> <span data-ttu-id="e41c7-145">既定のディレクトリは次のとおりです。</span><span class="sxs-lookup"><span data-stu-id="e41c7-145">The default directories are:</span></span>

| <span data-ttu-id="e41c7-146">OS</span><span class="sxs-lookup"><span data-stu-id="e41c7-146">OS</span></span>          | <span data-ttu-id="e41c7-147">パス</span><span class="sxs-lookup"><span data-stu-id="e41c7-147">Path</span></span>                          |
|-------------|-------------------------------|
| <span data-ttu-id="e41c7-148">Linux/macOS</span><span class="sxs-lookup"><span data-stu-id="e41c7-148">Linux/macOS</span></span> | `$HOME/.dotnet/tools`         |
| <span data-ttu-id="e41c7-149">Windows</span><span class="sxs-lookup"><span data-stu-id="e41c7-149">Windows</span></span>     | `%USERPROFILE%\.dotnet\tools` |

<span data-ttu-id="e41c7-150">これらの場所は、そこにインストールされたグローバル ツールを直接呼び出せるように、SDK が最初に実行されるときにユーザーのパスに追加されます。</span><span class="sxs-lookup"><span data-stu-id="e41c7-150">These locations are added to the user's path when the SDK is first run, so Global Tools installed there can be called directly.</span></span>

<span data-ttu-id="e41c7-151">注: グローバル ツールは、マシン全体ではなく、ユーザー固有のものです。</span><span class="sxs-lookup"><span data-stu-id="e41c7-151">Note: the Global Tools are user-specific, not machine global.</span></span> <span data-ttu-id="e41c7-152">ユーザーに固有であることは、そのマシンのすべてのユーザーが使用できるグローバル ツールをインストールできないことを意味します。</span><span class="sxs-lookup"><span data-stu-id="e41c7-152">Being user-specific means you cannot install a Global Tool that is available to all users of the machine.</span></span> <span data-ttu-id="e41c7-153">このツールは、ツールがインストールされた各ユーザー プロファイルでしか使用できません。</span><span class="sxs-lookup"><span data-stu-id="e41c7-153">The tool is only available for each user profile where the tool was installed.</span></span>

<span data-ttu-id="e41c7-154">グローバル ツールは、特定のディレクトリにインストールすることもできます。</span><span class="sxs-lookup"><span data-stu-id="e41c7-154">Global Tools can also be installed in a specific directory.</span></span> <span data-ttu-id="e41c7-155">特定のディレクトリにインストールした場合は、ユーザーはコマンドが使用できることを確認する必要があります。それには、そのディレクトリをパスに含めるか、指定されたディレクトリでコマンドを呼び出すか、指定したディレクトリ内からツールを呼び出します。</span><span class="sxs-lookup"><span data-stu-id="e41c7-155">When installed in a specific directory, the user must ensure the command is available, by including that directory in the path, by calling the command with the directory specified, or calling the tool from within the specified directory.</span></span>
<span data-ttu-id="e41c7-156">この場合、.NET Core CLI によりこの場所が PATH 環境変数に自動的に追加されることはありません。</span><span class="sxs-lookup"><span data-stu-id="e41c7-156">In this case, the .NET Core CLI doesn't add this location automatically to the PATH environment variable.</span></span>
